# HOW_TO_WORK.md - AI 에이전트 작업 및 배포 가이드

이 문서는 AI 에이전트가 작업 시 발생할 수 있는 실수를 방지하고, 정확한 실무 처리를 위해 준수해야 할 **절대 원칙**을 담고 있습니다. 작업을 시작하기 전 반드시 이 내용을 숙지하십시오.

---

## 🚨 1. 고스트 저장(Ghost Save) 및 인코딩 방지
*   **원칙**: 파일 수정 시 AI 전용 도구 대신 **PowerShell 스크립트**를 사용하여 직접 저장한다.
*   **형식**: 반드시 **BOM 없는 UTF-8 ($false)** 형식을 사용한다.
*   **백업**: 수정 전 반드시 `_backups/` 폴더에 원본을 복사한다.

```powershell
$file = '파일경로'
$content = @"
(전체 코드)
"@
$utf8NoBom = New-Object System.Text.UTF8Encoding $false
[System.IO.File]::WriteAllText($file, $content, $utf8NoBom)
```

## 📂 2. 통합 수정을 통한 배포 (실수 방지 핵심)
*   **원칙**: 관련된 모든 파일(HTML, CSS, JS, Asset)을 **동시에** 수정하고 검증한다.
*   **누락 방지**: 작업 중 `git status`를 수시로 확인하여 수정된 파일이 커밋에서 누락되지 않도록 한다.
*   **단일 커밋**: 기능 단위로 수정된 모든 파일을 **하나의 커밋**으로 묶어 푸시한다. (조각 커밋 금지)

## 📋 3. 작업 프로세스 및 최종 보고
1.  **계획 수립**: `task.md`에 요청 사항을 세분화하여 기록한다.
2.  **전수 검토**: 작업을 마치기 전, 처음 요청한 내용 중 빠진 섹션이나 이미지가 없는지 대조한다.
3.  **동기화 확인**: 로컬 브라우저(`localhost`) 화면과 서버(`Git`)에 올라가는 결과물이 100% 일치하는지 확인한다.
4.  **최종 브리핑**: 사용자에게 보고 시, 수정된 파일 목록과 변경 내용을 명확히 기술한다.

## 🔄 4. 복구 대책 (Recovery)
*   실수나 오류 발생 시 1초 내에 복구할 수 있도록 `HOW_TO_RECOVER_기능명.md` 문서를 작업마다 생성한다.
