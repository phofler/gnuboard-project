# HOW_TO_LIB_RESOLUTION_FIX.md

이 문서는 섹션 관리자의 이미지 고화질 유지(해상도 개선) 로직에 대한 기술 서입니다.

## 1. 개요
기존 시스템은 관리자 화면의 미리보기 박스(약 200px) 크기를 기준으로 이미지를 자르고 저장하여, 실제 웹사이트(큰 화면)에서 이미지가 흐릿하게 보이는 문제가 있었습니다. 이를 해결하기 위해 미리보기 크기와 별개로 **저장 해상도**를 강제하는 로직을 도입했습니다.

## 2. 구현 상세

### 1) 해상도 강제 옵션 (`js/image_smart_manager.js`)
- `SmartImageManager.open()` 호출 시 `forceW`, `forceH` 인자를 추가했습니다.
- 자동 감지(`detectRatio`)보다 이 강제 옵션의 우선순위를 높게 설정하여, 화면의 DOM 크기가 작더라도 고해상도 정보가 이미지 매니저(`image_manager.php`)로 전달되게 합니다.

### 2) 스킨 기반 해상도 매핑 (`adm/write.php`)
- `openUnsplashPopup()` 실행 시, PHP 함수 `get_mc_skin_info($ms['ms_skin'])`를 실시간으로 참조합니다.
- 예: Style B(Wide) 스킨인 경우 1200x600, Style A인 경우 800x600 등 스킨에 최적화된 높은 해상도 값을 JS 변수 `skin_w`, `skin_h`에 할당하여 전달합니다.

## 3. 효과
- 사용자가 보는 편집 화면은 작더라도, '적용' 버튼을 누르면 서버에는 스킨 권장 크기의 선명한 이미지가 생성되어 비주얼 품질이 개선됩니다.
