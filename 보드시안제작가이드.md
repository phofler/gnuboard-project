# 그누보드 트렌디 보드 스킨 제작 가이드 (Master Guide)

본 문서는 매거진, 웹진 등 트렌디한 스타일의 게시판 스킨을 **기획부터 실전 배포(PHP 변환)까지 시행착오 없이** 제작하기 위한 통합 가이드라인입니다.

---

## 1. 제작 워크플로우 (Workflow)

효율적인 스킨 제작을 위해 다음 3단계 프로세스를 권장합니다.

1.  **HTML/CSS 프로토타이핑**: PHP 연동 없이 순수 HTML로 완벽한 레이아웃과 반응형을 구현합니다.
2.  **기능 및 레이아웃 검증**: 모달 작동, 폼 필드 누락, 버튼 링크 등을 검수합니다.
3.  **PHP 스킨 변환 (Converting)**: 검증된 HTML에 그누보드 코드를 이식합니다.

---

## 2. 디자인 패턴별 구현 핵심

### A. 매거진 스타일 (Masonry Grid)
*   **컨셉**: 이미지 중심의 핀터레스트 스타일.
*   **구현**: CSS `column-count: 3` 사용. 아이템(`article`)에 `break-inside: avoid` 필수.
*   **UX**: 리스트에서 바로 내용을 볼 수 있는 **레이어 팝업(Modal)** 권장.

### B. 웹진 스타일 (Split Layout)
*   **컨셉**: 텍스트 가독성 중심. 썸네일과 텍스트 1:1 또는 4:6 비율.
*   **구현**: `display: grid` 또는 `flex`. `:nth-child(even)`으로 지그재그 배치.
*   **UX**: 상세 페이지에서 긴 글을 읽을 때 메타 정보(작성자, 공유 등)가 따라오도록 **Sticky Sidebar** 활용.

---

## 3. 필수 준수사항 (Do's & Don'ts)

### 🔴 레이아웃 및 UX
1.  **서브 레이아웃 유지**:
    *   글쓰기 페이지(`write.html`)도 리스트와 동일한 **서브 비주얼(Hero) 및 브레드크럼**을 유지해야 합니다. 폼만 덩그러니 있으면 안 됩니다.
2.  **수정 버튼 노출**:
    *   리스트 화면에서도 관리자가 바로 수정할 수 있도록 **수정 버튼(✎)**을 제목 옆 등에 배치합니다.
3.  **모달 내비게이션 (Overflow 이슈)**:
    *   모달 콘텐츠에 `overflow: hidden`이 있다면 화살표 버튼은 반드시 **모달 박스 바깥**(`position: fixed`)에 배치해야 잘리지 않습니다.
4.  **공유 기능 (View Only)**:
    *   `Share Code`, `Copy Link` 등은 글쓰기 폼이 아니라 **상세 보기(View)**에서 자바스크립트로 구현하는 *자동 기능*입니다. 입력 필드를 만들지 마세요.

### 🔴 글쓰기 폼(Form) 필수 구조 (개발자가 가장 많이 놓치는 부분)
HTML 폼을 만들 때 다음 필드가 누락되면 그누보드에서 저장이 안 됩니다.

**1. Form 태그 속성**
*   `action`: `write_update.php` 경로 (PHP 변환 시 `$action_url`)
*   `method`: `post`
*   `enctype`: `multipart/form-data` (파일 업로드 시 필수)

**2. 필수 Hidden Input**
*   `<input type="hidden" name="w" value="">` (작성모드)
*   `<input type="hidden" name="bo_table" value="">` (게시판ID)
*   `<input type="hidden" name="uid" value="">`
*   `<input type="hidden" name="wr_id" value="">`

**3. 입력 필드 네이밍 (name 속성 엄수)**
*   **카테고리**: `name="ca_name"` (select 또는 input)
*   **제목**: `name="wr_subject"` (필수)
*   **내용**: `name="wr_content"` (필수, 에디터 사용 시 JS 연동 필요)
*   **링크**: `name="wr_link1"`, `wr_link2`...
*   **파일**: `name="bf_file[]"` (배열 형태 필수)

---

## 4. PHP 변환 팁 (Converting Tips)

프로토타입이 완성되면 다음과 같이 치환합니다.

*   **반복문**: 더미 아이템 9개 → `<?php for ($i=0; $i<count($list); $i++) { ?> ... <?php } ?>`
*   **링크**: `view.html` → `<?php echo $list[$i]['href']; ?>`
*   **이미지**: `<img>` 태그 → `<?php echo get_list_thumbnail(...) ?>` (썸네일 생성 함수 활용)
*   **드래그 앤 드롭**: JS로 받은 파일을 `input[name="bf_file[]"]`에 할당하는 로직 추가.

## 5. 폴더링 표준
```text
/theme/테마명/skin/board/
├── skin_magazine/
│   ├── list.skin.php (목록+팝업)
│   ├── write.skin.php (글쓰기)
│   ├── view.skin.php (상세보기)
│   └── style.css
└── skin_webzine/ ...
```

이 가이드를 따르면 디자인과 개발 간의 간극을 최소화하고, 즉시 사용 가능한 고품질 스킨을 제작할 수 있습니다.
