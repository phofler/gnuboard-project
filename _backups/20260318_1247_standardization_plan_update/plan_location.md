# 마스터 액션 플랜 (Master Action Plan: plan_location.md)

> **"효율적인 테마 제작을 위한 플랫폼화 및 모듈 표준화 프로젝트"**

## 🎯 최종 목표: 테마 자동 생성 엔진 (Ultimate Goal)
본 프로젝트의 최종 목적은 단순히 코드를 정리하는 것이 아니라, **시안(HTML/Mockup)을 분석하여 즉시 작동하는 그누보드 테마로 자동 변환하는 엔진**을 구축하는 것입니다.

우리가 공통 함수와 공유 라이브러리를 정의하는 근거는 다음과 같습니다:
1. **시안의 데이터화**: 공통 라이브러리가 정교해질수록 시안의 비주얼 요소를 테마 데이터(CSS 변수 등)로 즉시 맵핑할 수 있습니다.
2. **자동 조립**: 표준화된 스킨과 위젯 함수를 통해, 새로운 시안이 오더라도 로직 코딩 없이 조립만으로 테마를 생성합니다.
3. **생산성 극대화**: "분석 ➜ style.css 변수 등록 ➜ 표준 위젯 배치"로 이어지는 자동화 프로세스를 완성합니다.

---

## 1. 프로젝트 목표
- **샘플 사이트 모사 및 테마 제작**: [판낼/2/](http://localhost/main/%ED%8C%90%EB%84%AC/2/) 디자인을 기반으로 신규 테마 구축.
- **플랫폼화 (Platforming)**: 모든 핵심 관리 모듈을 공통 라이브러리(`extend/`) 기반으로 표준화하여 테마 제작 생산성 극대화.

---

## 2. 공통 라이브러리 활용 전략
기존 분석된 `extend/` 내 공통 함수를 모든 신규 모듈 작업에 강제 적용합니다.

- **UI 표준화**: `get_theme_lang_select_ui()`, `get_skin_select_ui()`, `get_premium_editor_ui()` 사용.
- **유틸리티 표준화**: `get_project_asset_url()`를 통한 자산 경로 통합.
- **디자인 상속**: 테마의 `style.css` 변수(`--color-*`, `--font-*`)를 사용하여 스킨 수정 없이 디자인 동기화.

---

## 3. 모듈별 표준화 작업 리스트

| 순번 | 모듈명 | 작업 내용 |
| :--- | :--- | :--- |
| 1 | **보드 최근글 스킨 관리** | [완료] 테마/언어별 식별코드(`tm_id`) 적용 및 카드형 스킨 선택기 도입. |
| 2 | **카피라이트 관리** | 푸터 정보의 중앙 관리화 및 공유 UI 라이브러리 적용. |
| 3 | **서브 디자인 관리** | 상단 비주얼 영역의 통합 라이브러리(`display_hero_visual` 등) 연동. |
| 4 | **보드 스킨 관리** | 게시판별 스킨 설정 UI의 표준화 및 프리미엄 에디터 연동. |
| 5 | **회사소개 관리** | 섹션 기반 데이터 구조(`{module}_group`, `{module}_items`) 통일. |

## 5. 핵심 유의 사항
- **절대 경로 준수**: 모든 파일 참조는 `C:\gnuboard` 기반의 절대 경로/매크로 사용.
- **백업 필수**: 작업 전 `_backups/` 폴더 내 날짜별 백업 수행.
- **BOM 없는 UTF-8**: 모든 파일 저장 시 인코딩 규칙 엄격 준수.
