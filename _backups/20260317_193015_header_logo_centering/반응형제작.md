# 반응형 제작 가이드 (Standardized Responsive Framework)

이 문서는 모든 스킨에 공통으로 적용되는 반응형 제작 표준 및 글로벌 토큰 사용법을 정의합니다.

## 1. 핵심 원칙 (Core Principles)
1. **Token First**: 개별 CSS에 하드코딩된 수치를 적지 않고, 테마의 글로벌 변수(--bp-*, --fs-*)를 사용합니다.
2. **Fluid Design**: clamp() 함수를 사용하여 브레이크포인트 사이에서도 폰트와 간격이 유동적으로 변하게 합니다.
3. **Center-focused Scaling**: 이미지는 폭이 줄어들 때 항상 중앙subject를 유지하며 크롭되도록 설정합니다.

## 2. 디자인 토큰 (Design Tokens)

### 브레이크포인트 (Breakpoints)
- --bp-lg: 1200px (Large Desktop)
- --bp-md: 992px (Tablet Landscape)
- --bp-sm: 768px (Tablet Portrait / Mobile Wide)
- --bp-xs: 576px (Mobile Standard)
- --bp-xxs: 480px (Mobile Small)

### 타이포그래피 (Typography - Fluid)
- --fs-hero-title: clamp(1.6rem, 8vw, 4.5rem)
- --fs-hero-desc: clamp(1rem, 3vw, 1.8rem)
- --fs-section-title: clamp(1.8rem, 5vw, 3.5rem)

## 3. 표준 유틸리티 클래스
이미지가 중앙을 중심으로 축소되게 하려면 다음 속성을 공통으로 사용합니다.
`css
.img-center-focus {
    object-fit: cover;
    object-position: center center;
    transform-origin: center center;
}
`

## 4. 작업 절차
1. 전역 CSS(	heme/kukdong_panel/style.css)에 토큰 정의.
2. 개별 스킨의 CSS에서 하드코딩된 폭(width: 1200px 등) 및 폰트 크기 제거.
3. 전역 토큰을 호출하여 미디어 쿼리 및 스타일 작성.
