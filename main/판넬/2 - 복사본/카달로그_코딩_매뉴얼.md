# 카달로그 SEO 코딩 매뉴얼 (Catalog SEO Coding Manual)

이 문서는 성우첨단패널의 카달로그 이미지(약 30여 장)를 검색 엔진 최적화(SEO)가 적용된 프리미엄 HTML 페이지로 변환하기 위한 표준 작업 지침입니다.

---

## 1. 작업 개요
- **목적**: 이미지 내의 텍스트와 데이터를 HTML 코드로 변환하여 검색 노출(SEO)을 극대화하고, 사용자에게 전문적인 정보를 제공함.
- **핵심 원칙**: 
  1. 수치와 용어의 **100% 정확성** 유지 (이미지 데이터 전수 추출)
  2. **시맨틱 마크업** 사용 (검색 로봇 친화적 구조)
  3. **BOM 없는 UTF-8** 형식 준수 (시스템 안정성)

---

## 2. 표준 HTML 구조 (Template)

각 제품 섹션은 아래의 구조를 반복하여 사용합니다.

### A. 인트로 세션 (Premium Intro)
```html
<div class="product-intro-premium reveal slide-up">
    <div class="intro-content">
        <span class="cate" style="color: var(--edge-color);">제품 카테고리 (영어)</span>
        <h3>제품명 <span class="en-title">(영문명)</span></h3>
        <p class="intro-desc">
            카달로그 최상단 핵심 요약 문구 (100% 원문 그대로)
        </p>
    </div>
    <div class="intro-image-wrap">
        <img src="이미지파일명.jpg" alt="제품 시공 사례 및 특징">
    </div>
</div>
```

### B. 특장점 섹션 (Features)
```html
<div class="content-box reveal slide-up">
    <div class="seo-text-block">
        <h4>1. 제품 특징 (Key Features)</h4>
        <p>제품 전체에 대한 기술적 요약 설명...</p>
    </div>
    <div class="feature-grid">
        <!-- 각 특징 박스 반복 -->
        <div class="feature-box">
            <h4><i class="아이콘클래스"></i> 특징 제목</h4>
            <p>카달로그 이미지 내의 상세 설명 텍스트 (원본 유지)</p>
        </div>
    </div>
</div>
```

### C. 제원 및 성능 섹션 (Tech Specs & Performance)
```html
<div class="content-box reveal slide-up">
    <h4 class="sub-h4">2. 제품 제원 (Technical Specifications)</h4>
    <table class="spec-table">
        <thead>
            <tr><th>구분</th><th>상세 내용</th></tr>
        </thead>
        <tbody>
            <tr><th>용도</th><td>데이터...</td></tr>
            <!-- 필요한 행 반복 -->
        </tbody>
    </table>
</div>

<div class="content-box reveal slide-up">
    <h4 class="sub-h4">3. 성능 데이터 (Performance Data)</h4>
    <div class="performance-list">
        <ul>
            <li><strong>핵심키워드:</strong> 상세 설명 텍스트...</li>
        </ul>
    </div>
</div>
```

---

## 3. 주요 CSS 클래스 가이드

이미 `style.css`에 정의된 클래스들을 활용합니다.

- `.product-intro-premium`: 인트로 레이아웃 (그리드 1:1.2)
- `.cate`: 카테고리 텍스트 (엣지 레드 적용됨)
- `.en-title`: 제목 옆의 작은 영문 텍스트
- `.seo-text-block`: 배경색이 들어간 텍스트 강조 박스
- `.feature-grid`: 특장점 4구역 배치용 그리드
- `.spec-table`: 제원용 표준 테이블 스타일
- `.catalog-viewer`: 최하단 카달로그 이미지 뷰어 (점선 테두리)

---

## 4. 작업 프로세스 (Step-by-Step)

1.  **데이터 추출**: 카달로그 이미지에서 텍스트와 제원표 수치를 텍스트 파일로 추출함.
2.  **이미지 준비**: 시공 사례 이미지와 카달로그 원본 이미지를 프로젝트 폴더에 저장함.
3.  **코드 작성**: 위 Template에 맞춰 HTML 코드를 조립함. (파일명: `product_XX.html`)
4.  **SEO 검수**: 
    - `<img>` 태그의 `alt` 속성에 제품 관련 키워드가 포함되었는가?
    - 핵심 수치가 `<table>` 내에 정확히 코딩되었는가?
5.  **최종 저장**: PowerShell을 사용하여 **BOM 없는 UTF-8**로 저장함.

---

## 5. 저장 방식 (필수 명령)

파일 저장 시 반드시 아래 PowerShell 방식을 사용해야 합니다.

```powershell
$utf8NoBom = New-Object System.Text.UTF8Encoding $false
[System.IO.File]::WriteAllText("경로\파일명.html", $content, $utf8NoBom)
```

이 지침을 따르면 30여 장의 카달로그를 빠르고 정확하게, 그리고 검색 결과 상단에 노출될 수 있는 고품질 코드로 제작할 수 있습니다.
