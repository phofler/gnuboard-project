# 🚨 AI 배포 작업 시 필수 체크리스트

이 워크플로우는 AI 에이전트가 파일을 누락하거나 단계별 수정을 놓쳐서 발생하는 '허술한' 배포를 방지하기 위해 존재합니다.

## 1단계: 수정 전 환경 준비
- [ ] 작업을 시작하기 전 `git status`로 현재 상태를 확인한다.
- [ ] 요청받은 모든 항목을 `task.md`에 세분화하여 기록한다.

## 2단계: 파일 수정 및 로컬 검증
- [ ] 수정 시 연관된 모든 파일(HTML, CSS, JS, Asset 등)을 함께 수정한다.
- [ ] 수정된 전체 내용을 `git diff`로 대조하여 오타나 누락이 없는지 확인한다.
- [ ] **Ghost Save 방지**: 반드시 PowerShell BOM-free UTF-8 방식을 사용하여 저장한다.

## 3단계: 통합 커밋 및 푸시
- [ ] `git add` 시 수정된 모든 파일을 한꺼번에 스테이징한다. (파일별 개별 커밋 지양)
- [ ] 커밋 메시지에는 수정된 모든 파일의 핵심 변경 사항을 구체적으로 적는다.
- [ ] `git push` 성공 후 다시 한번 `git status`로 남은 변경 사항이 없는지 확인한다.

## 4단계: 최종 보고
- [ ] 사용자에게 보고 시, 요청 사항 중 어떤 부분이 어떻게 반영되었는지 파일별로 브리핑한다.
- [ ] 사용자에게 '강력 새로고침(Ctrl + F5)'에 대한 안내를 포함한다.
