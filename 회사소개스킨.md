# 회사소개 플러그인 스킨 가이드

## 개요
회사소개 플러그인(`company_intro`)에서 사용하는 스킨 목록 및 특징을 정리한 문서입니다.
현재 **Dark Theme**를 기준으로 제작되었으며, 추후 Light Theme 확장 시 CSS 변수를 활용하여 대응합니다.

## 1. 회사개요 / 인사말 (Intro)
| 스킨명 | 구분 | 설명 | 특징 |
|---|---|---|---|
| **Type A** | 이미지 / 텍스트 | 이미지(좌) - 텍스트(우) | 가장 보편적인 레이아웃. 안정적인 구조. |
| **Type A-1** | 이미지 / 텍스트 (CEO) | CEO Message | Type A 기반에 CEO 서명, 인용구 등 프리미엄 요소 추가. |
| **Type A-2** | 이미지 / 텍스트 (비젼) | Vision (Left Image) | 미래지향적인 비전 이미지와 텍스트 배치. |
| **Type B** | 텍스트 / 이미지 | 텍스트(좌) - 이미지(우) | Type A와 대칭되는 구조. 교차 배치 시 활용. |
| **Type B-1** | 텍스트 / 이미지 (CEO) | CEO Message (Right) | Type B 기반에 세련된 타이포그래피와 여백 적용. |
| **Type B-2** | 텍스트 / 이미지 (비젼) | Vision (Right Image) | 비전/가치 전달을 위한 깔끔한 텍스트 중심 레이아웃. |
| **Type C** | 중앙정렬 | 상단 이미지 - 중앙 텍스트 | 시선을 중앙으로 모으는 몰입형 레이아웃. |
| **Type C-1** | 중앙정렬 (CEO) | CEO Message (Center) | 중앙 정렬된 CEO 이미지와 메시지. |
| **Type C-2** | 중앙정렬 (비젼) | Vision Center | Type C 기반, 비전/가치 강조형 중앙 정렬 스킨. |

## 5. CSS 디자인 시스템 (Skin Layer)
모든 스킨은 공통 CSS (`/theme/corporate/css/default.css`)의 `:root` 변수를 상속받아 **다크 모드**를 구현합니다.

### 주요 CSS 변수 활용
```css
:root {
    --text-color: #ffffff; /* 본문 텍스트 (다크) */
    --accent-color: #d4af37; /* 포인트 컬러 (골드) */
    --bg-dark: #1a1a1a; /* 배경색 */
}
```
라이트 버전 전환 시, 위 변수값만 변경하면 스킨의 색상도 자동으로 반전되도록 설계되었습니다.

## 6. 스킨 제작 및 테마 상속 가이드 (Standard Protocol)

### 1) 배경색 제어 정책 (Background Policy)
- **관리자 설정 무시**: `write.php`의 배경색 선택 기능은 UI에서 숨겨졌으며, 사용자가 임의로 지정한 색상은 무시됩니다.
- **테마 강제 상속**: 모든 회사소개 페이지는 **무조건 테마의 배경색(`var(--color-bg-dark)`)을 사용**합니다. 이는 사이트 전체의 톤앤매너 통일성을 보장하기 위함입니다.
- **개발자 참고**: `view.php`에서 `$bg_color` 변수는 DB 값이 아닌 테마 변수로 고정되어 있습니다.

### 2) 신규 스킨 제작 규칙 (New Skin Checklist)
스킨을 추가하거나 수정할 때는 다음 규칙을 엄수해야 합니다.

1.  **NO Hardcoding (하드코딩 금지)**
    *   ❌ `background-color: #000000;`
    *   ✅ `background-color: var(--color-bg-dark);`
2.  **Typography Inheritance (폰트 상속)**
    *   스킨 파일 내에서 `font-family`를 직접 지정하지 마십시오. 테마의 `default.css`가 정한 폰트를 따릅니다.
    *   제목에 한해 `var(--font-heading)`을 사용할 수 있습니다.
3.  **Accent Color (포인트 컬러)**
    *   모든 강조 색상은 `var(--color-accent-gold)`를 사용해야 합니다. 다른 색상 사용 시 디자인 일관성이 깨집니다.

> [!IMPORTANT]
> 이 가이드를 지키지 않은 스킨은 추후 테마 설정(다크/라이트 모드) 변경 시 디자인이 깨지거나 **이질적인 페이지**로 남게 됩니다. 반드시 규격을 준수해주세요.

## 7. 스킨 CSS 표준 가이드 (CSS Publishing Standard)

새로운 스킨을 추가할 때 반드시 지켜야 할 상세 CSS 규약입니다.

### 1) 레이아웃 (Layout)
모든 스킨은 **테마 표준 너비(`--spacing-container`)**를 상속받아야 하며, 중앙 정렬되어야 합니다.

```css
/* 기본 래퍼 클래스 */
.premium-skin-wrapper {
    width: 100%;
    /* 테마 변수 상속 (기본값 1200px fallback) */
    max-width: var(--spacing-container, 1200px); 
    margin: 0 auto;
    /* [NEW] 테마 표준 여백 상속 (기본값 60px fallback) */
    padding: var(--spacing-section, 60px) 20px;
    box-sizing: border-box;
}
```

### 2) 색상 변수 (Color Variables)
하드코딩된 색상 코드(`#000`, `#fff` 등) 사용을 금지합니다. 테마에서 제공하는 **CSS 변수**를 사용하여 다크모드와 라이트모드를 자동 지원해야 합니다.

| 역할 | 변수명 | 설명 |
|---|---|---|
| **배경색** | `var(--color-bg-dark)` | 스킨 전체 배경 (다크모드 시 어두운 색) |
| **기본 텍스트** | `var(--color-text-primary)` | 제목 및 주요 본문 텍스트 |
| **보조 텍스트** | `var(--color-text-secondary)` | 부가 설명, 날짜 등 연한 텍스트 |
| **강조 포인트** | `var(--color-accent-gold)` | **핵심 포인트 컬러 (Gold)**. 버튼, 강조선 등 |
| **테두리** | `var(--color-border)` | 구분선, 박스 테두리 색상 |
| **상하 여백** | `var(--spacing-section)` | **섹션 상하 패딩** (기본 60px). 모든 스킨 통일. |

```css
/* 예시 */
.my-new-skin h3 {
    color: var(--color-text-primary); /* O */
    /* color: #333; (X) 절대 금지 */
}
```

### 3) 이미지 처리 (Images)
이미지 교체 및 반응형을 고려하여 CSS 처리를 해야 합니다.

```css
/* 이미지 컨테이너 */
.img-box {
    width: 100%;
    overflow: hidden;
    border-radius: 12px; /* 부드러운 모서리 표준 */
}

/* 이미지 태그 */
.img-box img {
    width: 100%;
    height: auto;
    display: block;
    object-fit: cover; /* 비율 유지하며 꽉 채우기 */
}
```

### 4) 반응형 (Mobile)
768px 이하 모바일 환경을 고려해야 합니다.

```css
@media (max-width: 768px) {
    .premium-skin-wrapper {
        padding: 40px 20px; /* 모바일에서 여백 축소 */
    }
}
```
