# 테마 디자인 및 CSS 규격 가이드 (Theme Design & CSS Standards)

이 문서는 앞으로 제작할 그누보드 테마의 일관성을 유지하고, 작업 효율을 극대화하기 위한 디자인 시스템 및 CSS 작성 규칙을 정의합니다.

## 1. 테마스킨 제작 목표 (Strategic Objectives)

우리는 독창적이고 프리미엄한 웹사이트 제작을 위해 실장님의 지침 아래 다음과 같은 핵심 목표를 가지고 테마와 스킨을 제작합니다.

1.  **독립된 디자인 파일**: 테마스킨은 독립된 파일 구조를 가지며, 각 업종이나 컨셉에 맞는 독창적인 디자인을 제공합니다.
2.  **효율적인 데이터 관리**: 모든 데이터(메뉴, 이미지, 콘텐츠 등)는 전용 플러그인을 통해 효율적으로 관리하여 운영의 편의성을 극대화합니다.
3.  **디자인 중심의 확장 (Skins & Theme Sync)**: 독창적인 디자인 구현을 위해 필요시 플러그인 내부에 전용 스킨(skins)을 추가합니다. 이때, **스킨의 크기와 색상은 반드시 `default.css`의 디자인 토큰을 따르도록 설계**하여 테마의 일관성을 유지합니다.

### 🌓 테마 동기화 및 핵심 디자인 원칙 (Core Design Principles)

1.  **Sovereignty of default.css**: 플러그인에서 새로운 디자인 스킨을 추가하더라도, 모든 색상값과 크기 규격은 테마의 `default.css`에 정의된 변수를 상속받아야 합니다. (디자인의 주권은 테마에 있음)
2.  **Master Dark & Ultimate Light**: 다크/라이트 테마는 핵심 변수(`:root`) 시스템을 공유하여 일관성을 유도합니다.
3.  **No Color Pollution (색상 오염 방지)**: 배경색 위에 반투명 오버레이를 겹칠 때는 반드시 범위를 제한하여 전체 톤이 탁해지는 것을 막습니다.
4.  **Mobile-Body Integration**: 모바일 헤더와 바디의 색상을 동기화하여 시각적 단절을 제거합니다.
5.  **Negative Space (여백의 미)**: 프리미엄 에디토리얼 디자인의 핵심은 '비움'에 있습니다. 인위적으로 공간을 채우려 하지 말고, 과감한 마진(Margin)과 패딩(Padding)을 사용하여 타이포그래피가 숨을 쉴 수 있는 공간을 확보합니다.
6.  **Variable Consolidation (변수 최소화 및 재사용)**: 색상과 크기가 같다면 반드시 동일한 전역 변수를 사용합니다. 플러그인마다 불필요한 전용 변수를 만드는 대신, 테마의 공통 토큰(`--color-accent`, `--spacing-section` 등)을 공유하여 관리 포인트를 단일화합니다. (DRY 원칙 준수)

### 🧩 변수 매핑 가이드 (Semantic Mapping)

| 구분 | 변수명 (추천) | 의미 및 용도 | 비고 |
| :--- | :--- | :--- | :--- |
| **색상** | `--color-accent` | 테마 전체의 핵심 강조색 (레드, 골든 등) | 플러그인별 고유 색상 지양 |
| | `--color-bg` | 사이트 전체의 기본 배경색 (#F3F3F3 등) | 헤더, 바디 등 공통 적용 |
| **간격** | `--mc-section-padding` | 메인 섹션의 상하 수직 여백 (100px 등) | 섹션 간 호흡(Rhythm) 통일 |
| | `--mc-item-gutter` | 이미지와 텍스트 사이의 간격 (50px 등) | 리스트 아이템 내부 정렬 |
| | `--spacing-container` | 컨텐츠의 최대 가로 너비 (1400px) | 모든 섹션 정렬 일치 |
| **타이포** | `--mc-title-size` | 메인 섹션 타이틀 크기 (3rem 등) | 섹션 헤더의 시각적 위계 |
| | `--font-heading` | 프리미엄 제목용 (Serif 등) | 테마의 인상을 결정하는 폰트 |

## 2. [Case Study] 레이어 겹침에 의한 배경색 왜곡 해결

실제 작업 과정에서 발생한 배경색 변색 문제를 통해 확립된 원칙입니다.

*   **현상**: 연회색 배경(#F3F3F3) 위에 메인 비주얼용 5% 블랙 오버레이가 화면 전체(100vh)를 덮으면서 무채색인 배경이 칙칙한 #E7E7E7로 변색됨.
*   **원인**: 오버레이 DIV가 이미지 컨테이너 외부(Global 범위)에 위치하여 배경색 전체에 간섭을 일으킴.
*   **해결책**: 오버레이 영역을 반드시 이미지/영상 부모 클래스(`hero_bg_wrap`) 내부로 한정하여 배경색의 순수성을 보존함.
*   **교훈**: 투명도가 있는 레이아웃은 반드시 독립된 컨테이너 내에서만 작동하도록 설계해야 함.

## 2. 테마 CSS 정의의 5가지 핵심 요소
테마 파일(`default.css`)에는 **"모든 페이지 공통"**이거나 **"브랜드 아이덴티티"**와 직결된 5가지만 정의합니다.

### 1) 디자인 토큰 (Design Tokens) - `변수 정의`
가장 중요합니다. 색상, 폰트, 여백 등을 변수(`:root`)로 정의합니다.
*   **Colors**: `--primary-color`, `--accent-color`, `--bg-dark`, `--text-main`
*   **Header**: `--color-header-bg`, `--color-header-border` (상단 바 레이아웃 전용)
*   **Menu**: `--color-menu-bg` (메뉴 판/드롭다운 등 컴포넌트 전용)
    - *Tip: 레이아웃과 컴포넌트 변수를 분리하여 관리해야 나중에 세밀한 디자인 수정 시 혼선이 없습니다.*
*   **Fonts**: `--font-heading` (세리프/제목용), `--font-body` (샌세리프/본문용)
*   **Spacing**: `--container-width` (1400px), `--header-height` (90~100px)

### 2) 타이포그래피 (Typography) - `글꼴 규칙`
`<h1>` ~ `<h6>`, `<p>`, `<a>` 태그의 기본 생김새입니다.
*   "모든 제목은 굵은 고딕체 (`Noto Sans KR` Bold)를 쓴다."
*   "모든 본문 줄간격(`line-height`)은 1.6이다."
*   "모든 링크(`a`)는 hover시 밑줄이 사라지고 색이 변한다."

### 3) 레이아웃 뼈대 (Layout Structure) - `공통 구조`
페이지라면 무조건 존재하는 영역들의 위치입니다.
*   **Header (#hd)**: 로고, GNB 메뉴의 위치. (배경색/보더는 변수 상속)
    *   **Context-Aware**: 메인(Floating/Transparent)과 서브(Solid/White) 구분을 지원하도록 설계.
*   **Wrapper & Container**: 전체 내용을 감싸는 폭과 중앙 정렬.
*   **Sub Visual (Hero)**: "회사소개", "제품소개" 같은 상단 큰 이미지 영역의 **높이(height)**와 **제목 위치**. (페이지 간 이동 시 덜컹거림 방지)
*   **Footer (#ft)**: 하단 정보 영역.

### 4) 공통 UI 컴포넌트 (Components) - `자주 쓰는 것들`
어느 페이지에서나 쓸 수 있는 버튼, 박스 등입니다.
*   **Buttons**: `.btn-primary` (메인 버튼), `.btn-outline` (테두리 버튼).
*   **Section Title**: 본문 중간중간 나오는 소제목 스타일 (`section h2`).
*   **Breadcrumb**: "Home > 회사소개 > 인사말" 경로 표시 스타일.

### 4) 통합 버튼 시스템 (Unified Button System) - `mc-btn`
모든 페이지의 CTA(Call-to-Action) 버튼은 개별 스타일이 아닌 공통 시스템을 따릅니다.
*   **.mc-btn**: 기본 뼈대 (여백, 자간, 트랜지션).
*   **.mc-btn-primary**: 배경색이 채워진 강조형 (Solid).
*   **.mc-btn-outline**: 테두리만 있는 세련된 타입 (Outline).
*   *Tip: 버튼의 크기와 자간은 `--mc-btn-padding`, `--mc-btn-letter-spacing` 변수로 관리합니다.*

### 5) 유틸리티 (Utilities) - `도우미 클래스`
*   `.text-center`, `.mt-50` (마진 탑 50px), `.pc-only` (PC에서만 보임) 등.

## 3. 디자인 토큰 (Design Tokens) - 실제 코드 예시
### Color Palette (Dark Default)
```css
:root {
    /* Base Colors */
    --bg-primary: #121212;   /* 메인 배경 (Dark) <-> (Light: #ffffff) */
    --bg-secondary: #1e1e1e; /* 서브 배경 (Card) <-> (Light: #f5f5f5) */
    
    /* Text Colors */
    --text-primary: #ffffff; /* 기본 텍스트 <-> (Light: #333333) */
    --text-muted: #888888;   /* 보조 텍스트 */
    
    /* Brand Colors */
    --brand-gold: #d4af37;   /* 포인트 컬러 (유지) */
    --brand-blue: #0044cc;   /* 링크/버튼 (유지 or 조정) */
}
```

## 4. 테마 구조 분석 (기존 내용)
그누보드 5 테마는 기본적으로 다음과 같은 파일 실행 순서를 가집니다:
1.  **`head.sub.php`**: HTML `head` 섹션. CSS 및 JS 라이브러리 로드.
2.  **`head.php`**: 웹사이트 상단(Logo, GNB) 및 서브 페이지 레이아웃(Hero Section).
3.  **`index.php`** (메인) 또는 **`skin/*`** (게시판/페이지): 실제 콘텐츠 본문.
4.  **`tail.php`**: 웹사이트 하단(Footer).
5.  **`tail.sub.php`**: HTML 하단 마무리 및 추가 스크립트.

### CSS 로드 방식
- `add_stylesheet('<link rel="stylesheet" href="'.G5_THEME_CSS_URL.'/파일명.css">', 우선순위);` 함수를 사용하여 `head.php`에서 로드하는 것을 권장합니다.

## 5. 테마 골격 (HTML Skeleton) 핵심 역할

테마의 일관성을 유지하기 위해 `#wrapper`, `#container_wr`, `#container`의 역할을 엄격히 구분합니다.

| ID | 명칭 | 핵심 역할 | 디자인 전략 |
| :--- | :--- | :--- | :--- |
| **`body`** | 시스템 바탕 | 사이트 전체 기본 배경 | `--color-bg-dark` 등 테마의 가장 근본 색상 정의. |
| **`#wrapper`** | 디자인 스위치 | 메인/서브 모드 전환 | `_INDEX_` 상수에 따른 **`.sub-layout-light`** 클래스로 전체 톤 제어. |
| **`#container_wr`** | 레이아웃 허브 | 본문 영역 전체 (Full) | 100% 가로너비 배경색/이미지 배치. |
| **`.sub-layout-width-height`** | **표준 가이드 (Hub)** | **컨텐츠 가이드 & 간격** | `max-width: 1200px`, `padding: 60px 20px 50px`로 **정렬 & 여백 통일**. |

> [!IMPORTANT]
> **효율적인 모드 전환 (`_INDEX_` 활용)**
> ```php
> <div id="wrapper" <?php echo (defined('_INDEX_')) ? '' : ' class="sub-layout-light"'; ?>>
> ```
> 위와 같이 `#wrapper`에서 메인/서브를 구분하면 하위 모든 자식 요소의 스타일을 한 번에 제어할 수 있어 유지보수가 매우 유리합니다. 이를 **'디자인 스위치'** 전략이라 합니다.

## 6. 'Corporate' 테마 분석 및 규칙 (Current State)

현재 작업 중인 `corporate` 테마는 **Premium Dark** 컨셉을 유지하며, 다음과 같은 분리 구조를 따릅니다.

### CSS 파일 분할 규칙 (4-File Structure)
- **`default.css`**: 글로벌 디자인 토큰(:root), 리셋, 기본 타이포그래피, 헤더/푸터 공통 스타일.
- **`sub_common.css`**: 서브 페이지 공통 레이아웃 (Hero Section, Breadcrumb).
- **`sub_style.css`**: 서브 페이지 내부 콘텐츠 (테이블, 폼, 버튼 상세 스타일).
- **`mobile.css`**: 모든 미디어 쿼리 및 모바일 전용 UI 스타일.

### 디자인 토큰 (Core Variables)
반드시 `default.css`의 `:root`에 정의된 변수를 사용하여 스타일을 입혀야 합니다.

| 변수 분류 | 변수명 | 기본값 (Corporate) | 용도 |
| :--- | :--- | :--- | :--- |
| **Colors** | `--color-bg-dark` | `#121212` | 전체 배경색 |
| | `--color-bg-panel` | `#1e1e1e` | 섹션/카드 배경 |
| | `--color-accent-gold` | `#d4af37` | 포인트 컬러 (Gold) |
| **Fonts** | `--font-heading` | `'Playfair Display'` | 주요 제목용 세리프체 |
| | `--font-body` | `'Inter'` | 본문용 샌세리프체 |
| **Size** | `--spacing-container`| `1400px` | 최대 콘텐츠 너비 |
| | `--header-height` | `100px` | 헤더 고정 높이 |

## 7. 새로운 테마 제작을 위한 CSS 규칙 (Guidelines)

앞으로 다른 디자인의 테마를 만들 때도 다음 규칙을 준수하여 효율성을 높입니다.

### 1) No Hardcoding (변수 사용 필수)
색상, 폰트 크기, 여백 등을 직접 입력하지 마십시오. 새로운 테마를 만들 때는 변수 값만 바꿔도 전체 분위기가 바뀌도록 설계합니다.

### 2) BEM 네이밍 규칙 준수 (Block-Element-Modifier)
클래스명 충돌을 방지하기 위해 BEM 방식을 권장합니다.
- `block`: `.hero-section`
- `element`: `.hero-section__title`, `.hero-section__image`
- `modifier`: `.hero-section--light`, `.hero-section--full`

### 3) Mobile First & Responsive
- 모든 미디어 쿼리는 한 파일(`mobile.css`) 또는 각 컴포넌트 하단에 모아서 관리합니다.
- 부트스트랩 등 외부 프레임워크 사용 시 테마 커스텀 변수와 충돌하지 않도록 주의합니다.

### 4) Sub Page Hero & Breadcrumb 표준화
- `head.php`에서 `sub_design` 플러그인 데이터를 받아 Hero 이미지를 출력하는 로직을 표준으로 사용합니다.
- 브레드크럼(Home > Depth 1 > Depth 2)은 테마 상관없이 동일한 HTML 구조를 유지하고 CSS로만 디자인을 변경합니다.

## 8. 효율적인 CSS 관리 전략 (Hub & Spoke Model)

테마가 늘어남에 따라 CSS가 스파게티처럼 꼬이는 것을 방지하기 위해 '허브 앤 스포크' 모델을 채택합니다.

### 1) Hub (Central Brain): `default.css`
- **역할**: 테마의 전체 생상, 폰트, 공통 규격 정의.
- **포함 내용**: `:root` 변수, Reset, Global Typography, Header/Footer Layout.
- **관리 원칙**: 모든 디자인 변경은 이 파일의 `:root`에서 시작됩니다.

### 2) Spoke (Specialized Styles): Plugin/Skin CSS
- **역할**: 특정 기능(회사소개, 게시판 리스트 등)에 특화된 레이아웃 정의.
- **관리 원칙**: 
    - 색상/폰트값을 직접 적지 않고 Hub의 변수를 호출합니다.
    - 예: `color: var(--color-accent-gold);`
    - 플러그인 스킨 내부의 `<style>` 태그는 최소화하고, 가능하다면 별도의 `.css` 파일로 분리하여 `add_stylesheet`로 로드합니다.

### 3) 공통 래퍼(Wrapper) 활용
- 모든 서브 페이지는 `#wrapper.sub-layout-light`라는 공통 부모를 가집니다.
- 이를 활용해 서브 페이지 본문 내의 제목(`h1~h6`), 문단(`p`), 링크(`a`)의 기본 스타일을 한 번에 정의하여 코드 중복을 제거합니다.
- 예: `#wrapper.sub-layout-light section h2 { ... }`

### 4) 캐시 버스팅 (Cache Busting)
- CSS 수정 후 즉시 반영되도록 그누보드의 `$G5_CSS_VER` 변수를 활용하거나, 로드 시 `time()` 함수를 파라미터로 붙여 관리합니다.

## 11. 신규 스킨 제작 가이드 (New Skin Development Protocol)
새로운 스킨을 추가할 때는 다음 체크리스트를 반드시 통과해야 합니다.

### [체크리스트]
1.  **[ ] Theme Colors Only**: 색상은 절대 하드코딩하지 않았는가? (Hex/RGB 금지)
2.  **[ ] Font Inheritance**: 폰트 패밀리를 직접 지정하지 않고 상속(`inherit`)하거나 테마 변수(`--font-heading`)를 사용했는가?
3.  **[ ] No Global Styles**: CSS 클래스명이 너무 일반적이지 않은가? (Bad: `.container`, Good: `.ci-container`)
4.  **[ ] Dark Mode Ready**: 배경색은 `--color-bg-dark`, 글자색은 `--color-text-primary`를 사용하여 테마 변경 시 자동 대응 되는가?

> [!WARNING]
> **하드코딩 절대 금지 (Strict Prohibition)**
> ```css
> /* ❌ WRONG */
> .my-skin { background: #000; color: #fff; font-family: 'Gulim'; }
> 
> /* ✅ RIGHT */
> .my-skin {
>     background: var(--color-bg-dark);
>     color: var(--color-text-primary);
>     font-family: var(--font-body);
> }
> ```

## 12. 관리 도구별 분석 및 주의사항 (Analysis by Manager)

현재 구축된 **8대 핵심 관리 모듈**을 기준으로, 테마스킨 제작 시의 역할과 주의사항을 정의합니다.

### 1) 상단메뉴스킨관리 (`top_menu_manager`)
*   **핵심**: 테마 컨셉에 맞는 GNB 디자인 선택 및 자동 렌더링.
*   **우리 방식**: 테마의 `head.php`에서 `display_pro_menu('스킨명')`으로 호출합니다.
*   **⚠️ 주의**: 메뉴 데이터는 DB에서 관리되므로 스킨 파일 내 하드코딩을 절대 금지합니다.

### 2) 메뉴관리 (`pro_menu_manager`)
*   **핵심**: 다국어 및 대형 메뉴 구조(Mega Menu)의 계층적 관리.
*   **우리 방식**: 트리 구조로 메뉴를 구성하며, 각 테마는 이 데이터를 바탕으로 독창적인 레이아웃을 입힙니다.
*   **⚠️ 주의**: 메뉴 코드(`me_code`)는 서브 비주얼과 콘텐츠 매칭의 핵심 키값이므로 삭제/변경 시 주의가 필요합니다.

### 3) 메인 이미지 관리 (`main_image_manager`)
*   **핵심**: 고해상도 이미지/영상과 에디토리얼 타이포그래피의 결합.
*   **구조**: **"식별코드(ID)"** 기반의 그룹 관리 시스템. 국문(`main_ko`), 영문(`main_en`) 등을 독립적으로 구성.
*   **호출**: 테마의 `index.php`
    - **표준 호출 규격**: `display_main_visual('식별코드')`
    - 식별코드를 비워둘 경우(`display_main_visual()`), 현재 활성화된 테마명과 동일한 ID가 있는지 먼저 찾고, 없으면 `default`를 출력합니다. (저작권 관리와 동일한 자동 매칭 로직)
*   **⚠️ 주의**: `ultimate_hero` 스킨처럼 오버레이가 포함된 경우, 오버레이 DIV를 반드시 이미지 컨테이너(`hero_bg_wrap`) 내부에 배치하여 테마의 순수한 배경색을 침범하지 않도록 합니다.

### 4) 메인 컨텐츠 관리 (`main_content_manager`)
*   **핵심**: 메인 페이지의 '섹션(Section)' 단위 동적 구성.
*   **규칙**: 각 섹션별로 스타일(Style A, B, C 등)을 선택할 수 있으며, 테마의 그리드 시스템(1400px)에 맞춰 렌더링됩니다.

### 5) 서브디자인관리 (`sub_design`)
*   **핵심**: 각 메뉴별 맞춤형 서브 비주얼(Hero) 매칭.
*   **규칙**: 메뉴 코드 기반으로 배경 이미지를 불러오며, 페이지 이동 시 시각적 일정한 톤을 유지하게 합니다.

### 6) 회사소개 관리 (`company_intro`)
*   **핵심**: 조직도, 연혁, 인사말 등 정형화된 기업 정보의 시각화.
*   **규칙**: 모든 데이터는 변수화되어 관리자에서 수정 가능하며, 테마의 디자인 토큰을 상속받습니다.

### 7) 온라인 문의 관리 (`contact_form`)
*   **핵심**: 비즈니스 로직(문의 접수)과 디자인의 완전한 분리.
*   **규칙**: 폼 필드와 정보({CP_TEL} 등)는 중앙 관리되며, 테마는 레이아웃 틀만 제공합니다.

### 8) 저작권 관리 (`copyright_manager`)
*   **핵심**: 하단 푸터 기업 정보 및 SNS 링크의 통합 관리.
*   **규칙**: 모든 하단 텍스트는 이 모듈에서 관리되며, 테마는 디자인 스타일만 결정합니다.
*   **[신규 스킨 추가 가이드]**:
    1.  **폴더 생성**: `plugin/copyright_manager/skins/style_x` 폴더 생성.
    2.  **필수 파일**:
        - `template.html`: 관리자 에디터 로드용 (CSS가 포함된 HTML).
        - `footer.skin.php`: 실제 프론트엔드 출력용 PHP.
        - `style.css`: 스킨 전용 CSS.
    3.  **치환자 활용**: `{logo}`, `{slogan}`, `{addr}`, `{tel}`, `{email}`, `{copyright}`, `{link1}`, `{link2}` 등의 치환자를 사용하여 데이터와 디자인을 분리합니다.
    4.  **연락처 링크**: 전화번호와 이메일은 반드시 클릭 시 연결되도록 `<a href="tel:{tel}">` 및 `<a href="mailto:{email}">` 형식을 사용합니다.
    5.  **관리자 등록**: `adm/write.php`의 `skin-list` 그리드에 라디오 버튼을 추가하고, `load_template()` JS 함수에 해당 스킨의 기본 컬러를 매핑합니다.

### 3) 작업 절차 (Workflow)
1.  **테마 폴더 생성**: 기존 테마를 복사하거나 새 폴더를 생성합니다.
2.  **디자인 토큰 수정**: `default.css`의 `:root` 변수를 해당 테마 컨셉에 맞춰 수정합니다.
3.  **레이아웃 조정**: `head.php`, `index.php` 등에서 테마 고유의 레이아웃 구조를 잡습니다.
4.  **플러그인 스킨 연결**: 각 관리 모듈에서 해당 테마에 어울리는 **공용 플러그인 스킨**을 호출하도록 설정합니다.

### 4) 안전 및 효율성 원칙 (Stability & Efficiency)
*   **기존 테마 보존**: 새로운 테마 작업이나 플러그인 스킨 수정 시, 기존에 운영 중인 테마(예: `corporate`)의 디자인이나 기능이 무너지지 않도록 반드시 상호 검증을 거쳐야 합니다.
*   **기록을 통한 효율화**: 모든 테마 구조와 플러그인 연동 방식은 문서화하여, 다음 테마 제작 시 시행착오를 줄이고 작업 효율을 극대화할 수 있도록 합니다.
