# Theme Sync & Plugin Standardization Manual

작성일: 2026-03-17

본 문서는 새로운 가비아/그누보드 테마를 제작할 때, 플러그인(회사소개, 메인 콘텐츠, 온라인 문의 등)의 색상과 스타일을 테마와 완벽하게 동기화하기 위한 표준 절차를 정리한 매뉴얼입니다.

---

## 0. 디자인(시안) 정밀 분석 (Analysis First)

테마 작업을 시작하기 전, 최상위 원칙은 **"시안이 모든 값의 기준이 된다"**는 것입니다. 임의의 색상을 사용하지 말고, 제공된 시안(HTML/CSS/이미지)을 분석하여 전역 변수값을 먼저 확정해야 합니다.

### 분석 절차
1.  **시안 폴더 탐색**: `main/시안명/` 폴더 내의 `index.css` 또는 `style.css`를 확인합니다.
2.  **Hex Code 추출**: 로고 색상(Brand), 강조 색상(Accent/Edge), 배경색(Secondary/Light)의 정확한 Hex Code를 추출합니다.
3.  **전역 변수(Global Variables) 동기화**: 추출한 값을 테마의 `style.css` 내 `:root` 섹션에 반영합니다. **이 과정이 선행되지 않으면 플러그인 동기화는 의미가 없습니다.**

## 1. 배경색(Background Color) 동기화 로직

모든 플러그인의 관리자 설정에서 배경색을 입력하지 않았을 때, 테마의 스타일을 자동으로 따라가도록 설계해야 합니다.

### 탐색 우선순위 (Variable Search Order)
1.  `--bg-white` (테마의 기본 화이트 배경)
2.  `--bg-light` (연한 회색 배경)
3.  `--color-bg` (테역 전용 배경 변수)
4.  **Fallback**: `#ffffff` (모든 탐색 실패 시 화이트로 수렴)

### 코드 구현 예시 (`adm/write.php`)
```php
$theme_bg_default = get_theme_css_value($config['cf_theme'], array('--bg-white', '--bg-light', '--color-bg'), '#ffffff');
```

## 2. 브랜드 및 엣지색(Branding Colors) 동기화

플러그인에 하드코딩된 특정 색상을 제거하고, **시안 분석을 통해 확정된 테마의 전역 변수**를 참조하게 합니다.

### 주요 포인트 색상 매핑
- **Primary Color**: `--color-brand` (#003366 - 시안 표준 딥블루)
- **Accent Color**: `--color-accent` (#004d99 - 시안 표준 액센트 블루)
- **Edge Color**: `--edge-color` (#c92127 - 브랜드 레드 포인트)
- **BG Light**: `--bg-light` (#f5f5f5 - 시안 표준 연회색)

### 스킨 파일 수정 가이드 (`.html` 또는 `.php`)
```css
/* BAD: 하드코딩 */
.highlight { color: #FF3B30; }

/* GOOD: 테마 변수 참조 */
.highlight { color: var(--edge-color, #c92127); }
```

## 3. 작업 시 절대 준수 사항 (Safety Rules)

### 🚨 Rule 0: 전역 백업 필수
작업 시작 전 `_backups/[날짜_시간_기능명]` 폴더를 생성하고 원본 파일을 보관한다.

### 🚨 Rule 7: 고스트 저장(Ghost Save) 방지
파일 수정 시 AI 도구의 단순 교체 대신, PowerShell을 사용하여 물리 디스크에 직접 쓰기를 강제한다.

### 🚨 Rule 2: BOM 없는 UTF-8 준수
PHP 레이아웃 깨짐과 AJAX 에러 방지를 위해 반드시 `BOM 미포함 ($false)` 설정을 사용한다.

## 4. 차세대 테마 생성 지침 (New Theme Standards)

`basic` 테마를 복사하여 새로운 프로젝트를 시작할 때 반드시 준수해야 할 구조적 설계 원칙입니다.

### [1] CSS/자산 독립화 및 경로 정규화 (Inheritance Rule)
*   **파일 복사**: `theme/basic/css/default.css`를 새 테마 폴더(`/css/`)로 반드시 복사하여 코어 의존성을 제거한다.
*   **경로 수정**: `head.sub.php`에서 모든 리소스를 테마 전용 변수로 호출한다.
    *   CSS: `<?php echo G5_THEME_CSS_URL ?>/파일명.css`
    *   IMG: `<?php echo G5_THEME_IMG_URL ?>/파일명.png`
*   **파일 위치**:
    *   `style.css` -> `/css/style.css` (테마 루트가 아닌 전용 폴더 관리)
    *   이미지 -> `/Img/` 또는 `/img/` 폴더 (루트 이미지 방치 금지)

### [2] 레이아웃 초기화 (Layout Reset)
*   시안 작업 전, `default.css`의 구형 해상도 설정(`width: 930px`, `float: left` 등)을 제거한다.
*   중앙 정렬은 `margin: 0 auto` 및 `display: flex`를 활용하여 현대적인 시안 구조로 재정의한다.

### [3] 지능형 지침서 결합
*   테마 생성 즉시 **`HOW_TO_LIB_테마명.md`**를 생성하여, 테마 전용 로직과 복구 가이드를 단일 문서로 통합 관리한다.

## 5. 테마 자산 구조 표준 (Asset Structure)

```text
theme/your_theme/
├── css/
│   ├── default.css      (코어 복사본, 뼈대 수정용)
│   └── style.css        (메인 디자인, 옷 입히기용)
├── Img/                 (모든 이미지 자산)
├── js/                  (테마 전용 스크립트)
├── skin/                (게시판 및 플러그인 스킨)
├── head.sub.php         (CSS/JS 경로 정의)
└── theme.config.php     (테마 설정)
```
