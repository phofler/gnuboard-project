# 테마스킨 제작 가이드 (Theme Skin Development Guide)

> **🎯 핵심 목표 (Primary Goal)**
> 1. **신속한 납품**: 메인/서브 시안 
>    `/main/corporate_light/index_ko.html`
>    `/main/corporate_light/sub_ko.html`
>    스킨 작업 후 웹서버 등록.
> 2. **효율성**: 무작정 코딩 금지. **시안(index/sub)을 분석하여 ➜ 테마의 `default.css`에 전역 변수로 등록하고 ➜ 이를 통해 스킨을 효율적으로 작업한다.**
> 3. **우선순위**: **매뉴얼 정리 > 코딩 작업.** ("매뉴얼이 완벽하지 않으면 코드는 쓰레기다.")

---

이 문서는 그누보드 테마 및 플러그인 스킨의 일관성을 유지하고, 프리미엄한 디자인 품질을 보장하기 위한 공식 제작 표준을 정의합니다. 모든 제작자는 본 가이드를 준수하여 '테마 주권'이 확립된 환경에서 스킨을 제작해야 합니다.

---

## 🏛️ PART 1. 공통 원칙 및 디자인 시스템 (Foundation)

테마의 뼈대가 되는 철학, 변수, 그리고 구조에 대한 정의입니다. 코딩 전 반드시 숙지해야 합니다.

### 1. 제작 철학 및 핵심 원칙 (Core Principles)

#### 1) 테마 주권 (Theme Sovereignty)
플러그인이나 개별 스킨은 '구조(Structure)'만 결정하며, 모든 '시각적 속성(Property: 색상, 폰트, 여백)'은 테마의 `default.css`가 결정합니다.
*   **Zero Tolerance (무관용 원칙)**: 스킨 파일(`latest.skin.php` 등) 내부에 `<style>` 태그로 색상(`#000`)을 직접 박는 행위를 엄격히 금지합니다.
*   **Solution**: 반드시 `var(--color-bg-dark)`와 같이 테마 변수를 호출하십시오.

#### 2) 비움의 미학 (Negative Space)
프리미엄 디자인의 핵심은 '비움'에 있습니다. 과감한 마진(Margin)과 패딩(Padding)을 사용하여 타이포그래피가 숨을 쉴 수 있는 공간을 확보하십시오.

#### 3) 색상 오염 방지 (No Color Pollution)
배경색 위에 반투명 오버레이를 사용할 경우, 반드시 특정 컨테이너 내부로 범위를 제한하여 전체 톤이 탁해지는 것을 방지합니다.

#### 4) 뷰 래퍼 프리 (View Wrapper-Free)
*   **원칙**: 플러그인의 `view.php`는 스킨을 위한 '액자' 역할만 해야 합니다. 배경색이나 여백이 필수적인 경우가 아니라면, **불필요한 `div`로 스킨을 감싸지 마십시오.**
*   **Why?**: 중복된 Wrapper(Div Soup)는 테마 CSS 제어를 방해합니다.

#### 5) 시각적 위계의 절대성 (Rule of Visual Authority)
*   **정답지**: HTML 시안(`/main/.../index_ko.html`)이 모든 디자인의 절대적 정답입니다.
*   **원칙**: 매뉴얼의 규칙과 시안의 비주얼이 충돌할 경우, **무조건 시안의 비주얼을 우선**합니다.

---

### 2. 디자인 시스템: 토큰 및 변수 (Design System)

모든 스타일 정의 시 하드코딩을 금지하며, 다음의 디자인 토큰을 참조해야 합니다.

#### 1) 핵심 색상 토큰
| 변수명 | 용도 | 비고 |
| :--- | :--- | :--- |
| `--color-bg-dark` | 사이트 전체 기본 배경색 | 라이트 모드 대응 필수 |
| `--color-brand-etsy` | `#F1641E` (Etsy Orange) | 라이트 테마 푸터 및 강조색 표준 |
| `--color-accent-gold` | 브랜드 핵심 강조색 | 에센셜 포인트용 |
| `--color-text-primary` | 기본 본문 텍스트 색상 | 가독성 중심 |
| `--color-text-secondary` | 보조 텍스트 및 설명글 | 투명도 기반 설정 권장 |

#### 2) 레이아웃 및 간격 토큰
| 변수명 | 표준값 (시안 기준) | 용도 |
| :--- | :--- | :--- |
| `--spacing-container` | **`1360px`** | 콘텐츠 최대 가로 너비 (1360px 엄수) |
| `--container-padding` | **`0` (PC) / `5vw` (Mobile)** | 반응형 패딩 정책 |
| `--spacing-section` | **`10vh`** | 섹션 간 수직 여백 |
| `--header-height` | `100px` | 상단 헤더 고정 높이 |

#### 3) 레이아웃 유틸리티 (Layout Utilities)
서브 페이지에서 콘텐츠 폭을 제어하기 위해 제공되는 표준 클래스입니다.
*   `.skin-container` (표준 1360px): 일반적인 게시판/스킨용
*   `.skin-container-1400` / `1500` / `1600`: 확장형 폭이 필요할 때 사용
*   `.skin-container-full`: 100% 폭 (배경 확장형)

---

### 3. 타이포그래피 표준 (Typography Standards)

#### 1) 섹션 타이틀 (Section Title & Edge Bar)
*   **메인 페이지**: 중앙 정렬, 에지 바 없음. `color: var(--color-title-main)`
*   **서브 페이지**: 좌측 정렬, **에지 바(Edge Bar) 필수**. `color: var(--color-title-sub)`
*   **Hero 영역**: 배경 이미지 위는 항상 **White**.

#### 2) 섹션 서브 타이틀
*   `.section-subtitle`: `var(--color-accent-gold)`, Uppercase, Font-weight 600.

---

### 4. 테마 및 스킨 아키텍처 (Architecture)

#### 1) 허브 앤 스포크 모델
*   **Hub**: `default.css` (모든 전역 변수 정의)
*   **Spoke**: 플러그인/스킨 CSS (변수 상속)

#### 2) 네이밍 규칙 (Namespace)
*   `.ci-` (Company Intro), `.mc-` (Main Content), `.theme-` (Theme Component)

#### 3) 서브페이지 레이아웃 전략
*   **Full Width**: 갤러리/제품 리스트 (컨테이너 해제)
*   **Sidebar (LNB)**: 정보성 페이지 (사이드바 포함)
*   **구조 표준**: `<aside class="sub-sidebar">` + `<main class="sub-content">`

## 🛠️ PART 2. 제작 공정 및 도구 (Workflow & Tools)

실제 스킨을 제작할 때 따르는 절차와 도구입니다.

### 5. 핵심 생산 공정 (The Core Process)

#### Step 1: 정밀 분석 (Analysis)
시안(`index_ko.html`)을 열어 반복되는 색상, 폰트, 간격, 레이아웃을 현미경처럼 분석합니다.

#### Step 2: 테마 전역화 (Globalizing)
분석된 요소를 `default.css`에 변수(`--color...`)와 클래스(`.hero-title`)로 등록합니다.
*   **Rule**: 시안의 변수명을 그대로 쓰지 말고 테마 표준 변수명으로 매핑하십시오.

#### Step 3: 효율적 스킨 조립 (Assembly)
등록된 클래스를 활용하여 스킨 파일(`php`)을 조립합니다. `<style>` 태그 작성을 최소화합니다.

---

### 6. 생산성 도구 (Production Tools)

#### 1) 타이포그래피 치트시트
*   메인 제목: `var(--color-title-main)`
*   서브 제목: `var(--color-title-sub)`
*   본문: `var(--color-text-secondary)`

#### 2) 5초 컷 스타터 킷 (Starter Kit)
모든 스킨 제작 시 기본으로 사용하는 보일러플레이트 코드입니다. (`skin.head.php` 포함)

```php
<?php
if (!defined('_GNUBOARD_')) exit;
include_once(dirname(__FILE__) . '/../skin.head.php');
$uniqid = uniqid('skin_');
?>
<section class="mc-skin-wrapper <?php echo $skin_cls; ?>" id="<?php echo $uniqid; ?>">
    <div class="skin-container">
        <div class="section-header text-center">
            <h2 class="section-title"><?php echo $txt_title; ?></h2>
            <p class="section-subtitle"><?php echo $txt_desc; ?></p>
        </div>
        <div class="skin-content">
            <!-- Content Here -->
        </div>
    </div>
</section>
```

---

### 7. 고급 개발 표준 (Advanced Standards)

#### 1) 레거시 블랙 필터링
`#000000` 색상이 들어오면 강제로 테마 기본값으로 치환하여 다크모드 호환성을 확보합니다.

#### 2) 디자인 격리 시스템 (Isolation)
프리미엄 스킨은 `.isolation-wrap`으로 감싸 테마의 글로벌 스타일 간섭을 차단합니다.

#### 3) 에디터 환경 안정화
`iframe` 내부에도 `default.css`를 주입하고, 배경색 강제 적용 로직을 통해 WSYIWYG 환경을 보장합니다.

#### 4) 이중 패딩 금지
`.container`에 이미 패딩이 있으므로, 내부 요소에 중복 패딩을 주지 마십시오.

---

### 8. 품질 검증 및 트러블슈팅 (QA)

#### 1) 변수 우선순위 금기
단순 Hex 값보다는 Brand Token, 그보다는 **Semantic Variable**을 우선 사용하여 유지보수성을 높입니다.

#### 2) 시안 완벽 동기화
`px` 실수, 자간/행간 누락, 섹션 누락 등을 방지하기 위해 시안과 1:1 대조 검수를 수행합니다.

#### 3) 레이아웃 트러블슈팅 (Layout Troubleshooting)
*   **증상**: 갤러리 게시판이 화면 끝까지 늘어남.
*   **원인**: 서브 페이지는 기본적으로 `width: 100%`로 풀려있음.
*   **해결**: 스킨 최상단에 `.skin-container` 클래스를 추가하여 폭을 잡아줍니다.

## 📂 PART 3. 스킨별 상세 제작 가이드 (Specifics by Menu)

관리자 메뉴 순서에 따른 각 모듈별 상세 구현 가이드입니다.

### 9. 관리자 메뉴별 매뉴얼

#### 1) 메인 이미지 관리 (Main Image Helper)
*   **용도**: 히어로 영역 (슬라이드, 영상)
*   **필수 클래스**: `.skin-mim`
*   **핵심 변수**: `var(--spacing-hero)` (높이), `var(--font-display)` (폰트)

#### 2) 상단메뉴스킨관리 (Top Menu)
*   **용도**: GNB 네비게이션
*   **필수 클래스**: `.skin-tmm`
*   **핵심 변수**: `var(--header-height)`, `var(--color-header-bg)`
*   **💡 핵심 패턴: 스티키 헤더 배경 전환 (Sticky Transition)**
    *   **의도**: 초기 투명(Transparent) 헤더의 개방감과 스크롤 시 고정(Fixed) 헤더의 가용성(가독성)을 동시에 확보.
    *   **구현**: JS로 `window.scrollY`를 감지하여 `.is-scrolled` 클래스를 토글하고, CSS에서 `var(--color-header-bg)`와 `box-shadow`를 적용함.
    *   **관리 유의사항**: 
        1. 디자인 로직은 스킨 파일(`menu.skin.php`, `style.css`)에 위치하므로, **소스 수정 시 관리자 페이지 [설정 저장] 버튼 클릭 없이 즉시 반영**됨.
        2. 관리자 페이지의 [설정 저장]은 '스킨 종류 변경'이나 '로고 이미지 교체' 등 DB 데이터 변경 시에만 사용함.

#### 3) 메인 컨텐츠 관리 (Main Content)
*   **용도**: 메인 페이지 섹션 위젯 (Story, Works 등)
*   **필수 클래스**: `.skin-mcm`
*   **핵심 변수**: `var(--spacing-section)` (수직 간격)
*   **주의사항**: 최신글 스킨은 테마 폴더(`/theme/skin/...`)에 위치시킵니다.

#### 4) 보드 스킨 관리 (Board Skin Standards)
**목표**: 스킨 제작 시 아래 4단계 프로세스를 순차적으로 적용하여 기능 누락을 원천 차단한다.

**(0) 공통 표준 및 스타일 가이드 (Common Rules & Style Guide)**
*   **식별자**: 폴더명은 `skin_타입명` (예: `skin_magazine`), 게시판 ID는 `corporate_타입명` 패턴 준수.
*   **Context 유지**: 모든 내부 링크(`href`)에 `me_code`(메뉴)와 `sca`(카테고리) 파라미터를 반드시 이어붙인다. (`$qstr` 활용 권장)
*   **Layout 상속**: 레이아웃 래퍼로 `.skin-container` (1360px) 사용 필수.
*   **Buttons 상속**: 자체 버튼 스타일 대신 `.mc-btn` (기본), `.mc-btn-primary`, `.mc-btn-outline` 클래스 사용 권장.
*   **Inputs 상속**: `default.css`에는 글로벌 폼 스타일이 없으므로, 스킨 내 `style.css`에서 테마 변수를 매핑해야 함.
    *   `input.frm_input { border: 1px solid var(--color-border); padding: 10px; }`
*   **Typography**: 폰트는 `var(--font-body)`를 상속받으므로 별도 지정 금지.

**(1) 리스트 페이지 (List Page)**
*   **레이아웃**: `Masonry` 또는 `Grid` 스타일을 기본으로 하며, 반응형(`mobile`) 처리를 우선한다.
*   **책갈피(Anchor)**: 관리자용 **[수정]**, **[글쓰기]** 버튼에는 링크 끝에 반드시 `#bo_w`를 붙여 입력 폼으로 직행시킨다.
    *   *Why?*: 수정 클릭 시 긴 스크롤을 다시 내려야 하는 불편 해소.
*   **필터링**: 카테고리 탭 클릭 시 `sfl/stx` 검색 방식으로 구현하여 매칭 정확도를 높인다.

**(2) 레이어 갤러리 (Layer Viewer - Premium)**
*   **Split Layout**: 프리미엄 스킨(Magazine 등)은 60(갤러리):40(정보) 스플릿 레이아웃을 기본으로 한다.
*   **UX**: 갤러리/매거진 스킨에서 이미지 클릭 시 페이지 이동 없이 **모달(Modal)**을 띄우는 것을 원장한다.
*   **Navigation**: 이전/다음 화살표는 모달 박스 **바깥(ViewPort 기준 Fixed)**에 배치하여 클릭 영역을 최대화한다.
*   **Context**: 모달 내부에 '상세보기(Read More)' 버튼을 두어 `me_code`가 포함된 개별 뷰 페이지로 연결한다.
*   **Edit in Modal**: 관리자 편의를 위해 모달 제목 옆에 수정 버튼(✎)을 배치한다.

**(3) 뷰어 페이지 (View Page / General)**
> [!TIP]
> **"Information Layering: Separate Data from Meta."**
> 뷰어 페이지는 단순히 글을 보여주는 것이 아니라, 정보의 우선순위를 시각적으로 분리하여 가독성을 극대화해야 합니다.

*   **Layout (Asymmetric Grid)**: 
    - 본문(`article-main`): 좌측 또는 중앙 배치 (최대 너비 700~800px로 제한하여 가독성 확보).
    - 메타 정보(`meta-sidebar`): 우측 **Sticky Sidebar** 배치. 작성일, 카테고리, 공유 버튼 등을 본문 스크롤에 관계없이 항상 노출.
*   **Typography Hierarchy**:
    - **Article Lead**: 본문 시작 전 커다란 폰트와 볼드, 혹은 인용구 처리(Quotation)를 통해 글의 핵심 요약을 강조.
    - **Sticky Meta**: 세리프체(Italic)와 산세리프체를 혼용하여 전문적인 잡지 분위기 연출.
*   **Wrapper Standard**: `view.skin.php`는 레이아웃용 `div`를 남발하지 않고 `skin-container` 하나로 정리하여 테마의 중심축을 유지한다.

**(4) 등록/수정 페이지 (Write Page)**
*   **Structure**: `<section id="bo_w" class="skin-container">` 구조를 엄수하여 테마 레이아웃과 충돌을 방지한다.
*   **Logic (JS)**: 다단계 분류가 필요할 경우, 인라인 스크립트 대신 표준 모듈 `theme/corporate_light/js/category_cascade.js`를 로드하여 사용한다.
*   **Flow (Redirect)**: `write_update.tail.skin.php`를 반드시 포함하여, 저장 후 **뷰가 아닌 목록(List)으로 이동**시킨다.
    - **중요**: 리다이렉트 시 `sca` 파라미터 대신 **`sfl=ca_name&stx=분류명`**을 사용하여 필터링 결과가 즉시 반영되도록 한다. (눈에 보이지 않는 공백/인코딩 이슈 방지)

### 3) 트러블슈팅: 카테고리 필터 결과 미출력 (Troubleshooting: 0 Results)
> [!WARNING]
> **"Query is correct, but results are 0?"**
> SQL 쿼리는 정상인데 게시물이 나오지 않는다면 다음 두 가지를 반드시 체크하십시오.

1.  **앵커(`#`)의 치명적 위치**:
    *   **오류**: `board.php?#bo_cate&sca=서울` (앵커가 파라미터 앞)
    *   **현상**: 브라우저는 `#` 이후를 서버로 보내지 않으므로 `sca` 값이 전달되지 않아 0건이 나옵니다.
    *   **해결**: 앵커(`#bo_cate`)는 반드시 **URL의 최하단**(`...&sca=서울#bo_cate`)에 위치시켜야 합니다.
2.  **데이터 매칭 정밀도 문제 (Exact vs Search)**:
    *   **오류**: 게시판 고유 기능인 `sca`(완전 일치)는 눈에 보이지 않는 공백 하나만 틀려도 매칭에 실패합니다.
    *   **해결**: 모든 분류 필터를 **검색 방식**(`sfl=ca_name&stx=분류명`)으로 전환하십시오. 그누보드 코어의 `INSTR` 매칭 로직을 타게 되어 공백이나 인코딩 오차에 관계없이 견고하게 결과가 노출됩니다.
3.  **데이터 정규화**:
    *   DB 내 `ca_name`에 줄바꿈이나 중복 공백이 없는지 확인하고, 스킨 내에서도 `preg_replace('/\s*>\s*/', ' > ', $cat)` 등을 통해 ` > `(앞뒤 한 칸 공백) 표준을 강제 적용하십시오.

### 6) 트러블슈팅: 카테고리 설정 오염 (Category Corruption)
*   **증상**: 화면상 카테고리 이름 뒤에 `11111` 등 알 수 없는 숫자가 붙어 나옴. (`RESIDENCE11111`)
*   **원인**: DB의 `bo_category_list` 설정값이 오염됨 (관리자 실수나 스크립트 오작동).
*   **응급 복구(Hotfix)**:
    1.  서버 루트에 `fix_category.php` 생성 (또는 기존 파일 활용).
    2.  아래 코드로 실행하여 강제로 표준 포맷(`Parent > Child`)으로 덮어쓰기.
    ```php
    // 예시: 표준 카테고리 리스트로 복구
    $clean_list = "RESIDENCE|RESIDENCE > 서울|RESIDENCE > 서울 > 은평구|...";
    sql_query(" update {$g5['board_table']} set bo_category_list = '{$clean_list}' where bo_table = '게시판ID' ");
    ```
*   **예방**: `bo_category_list`는 수동으로 건드리지 말고, 변경 시 반드시 백업할 것.

### 11. 작업 기록: 매거진 스킨 프리미엄 리뉴얼 (2026.01.14)
*   **작업 목표**: 
    1. 디자인 시스템 완전 통합 (`default.css` 변수 상속).
    2. 프리미엄 스플릿 뷰(60:40) 라이트박스 구현.
    3. `me_code` 및 검색 기반 필터링(`sfl=ca_name`) 표준 적용.
*   **반복 작업 및 트러블슈팅 (Repetitive Refinements)**:
    *   **Post-Write Visibility Fix**: 글 등록 후 `sca` 불일치로 인해 게시물이 사라지는 현상을 해결하기 위해, 리다이렉트 로직을 검색 기반(`sfl=ca_name&stx=...`)으로 전면 전환함.
    *   **Read More 삭제**: 대다수 사용자가 모달 자체를 하나의 완성된 상세 페이지로 인식하므로, 불필요하게 뷰 페이지로 유도하는 'Read More' 버튼을 반복적으로 삭제함. (표준: 모달만으로 정보 완결성 확보)
    *   **Author HTML Leak**: 그누보드의 `$list[$i]['name']`은 관리자 설정에 따라 아이콘/링크 등 HTML 태그를 포함하므로, JS 데이터 전달 시 `.innerText`가 아닌 `.innerHTML`을 사용해야 함.
*   **핵심 성과**:
    *   **Theme Sovereignty**: 자체 배경색을 버리고 `var(--color-bg-panel)`을 상속받아 테마 설정 변경에 유연하게 대응함 (Option 1 적용).
    *   **UX Consistency**: 수정/글쓰기 후 목록 복귀 시 카테고리 앵커(`#bo_cate`)를 사용하여 스크롤 피로도를 낮춤.
*   **Lesson**: 스킨 제작 시 로컬 변수를 남발하지 말고, 테마의 글로벌 토큰(Color, Font, Spacing)을 먼저 매핑하는 것이 유지보수의 핵심임.

### 4.3 취소 버튼 앵커링 (Cancel Button Anchoring)
- **규칙**: 글쓰기/수정 화면의 '취소' 버튼 클릭 시, 단순히 리스트로 돌아가는 것이 아니라 사용자가 작업하던 **카테고리 위치(#bo_cate)** 또는 **리스트 상단(#bo_list)**으로 스크롤을 이동시켜야 한다.
- **이유**: 사용자가 긴 스크롤 후 글을 쓰다가 취소했을 때, 다시 스크롤을 내리는 불편함을 방지하기 위함.
- **적용 방법**: `get_pretty_url`로 생성된 링크 뒤에 `#bo_cate`를 추가한다. `me_code` 유지도 필수.
