# 테마스킨 제작 가이드 (Theme Skin Development Guide)

> **🎯 핵심 목표 (Primary Goal)**
> 1. **신속한 납품**: 메인/서브 시안 
>    `/main/corporate_light/index_ko.html`
>    `/main/corporate_light/sub_ko.html`
>    스킨 작업 후 웹서버 등록.
> 2. **효율성**: 무작정 작업 금지. **매뉴얼(`테마스킨.md`) 선행 작업**을 통해 시행착오 최소화.
> 3. **우선순위**: **매뉴얼 정리 > 코딩 작업.**

---

이 문서는 그누보드 테마 및 플러그인 스킨의 일관성을 유지하고, 프리미엄한 디자인 품질을 보장하기 위한 공식 제작 표준을 정의합니다. 모든 제작자는 본 가이드를 준수하여 '테마 주권'이 확립된 환경에서 스킨을 제작해야 합니다.

---

## 1. 제작 철학 및 핵심 원칙 (Core Principles)

### 1) 테마 주권 (Theme Sovereignty)
플러그인이나 개별 스킨은 '구조(Structure)'만 결정하며, 모든 '시각적 속성(Property: 색상, 폰트, 여백)'은 테마의 `default.css`가 결정합니다.
*   **Zero Tolerance (무관용 원칙)**: 스킨 파일(`latest.skin.php` 등) 내부에 `<style>` 태그로 색상(`#000`)을 직접 박는 행위를 엄격히 금지합니다.
*   **Why?**: 스킨에 색이 박혀있으면, 나중에 테마를 'Light'에서 'Dark'로 바꿀 때 스킨을 일일이 열어서 수정해야 하는 비효율이 발생합니다.
*   **Solution**: 반드시 `var(--color-bg-dark)`와 같이 테마 변수를 호출하십시오.

### 2) 비움의 미학 (Negative Space)
프리미엄 디자인의 핵심은 '비움'에 있습니다. 과감한 마진(Margin)과 패딩(Padding)을 사용하여 타이포그래피가 숨을 쉴 수 있는 공간을 확보하십시오.

### 3) 색상 오염 방지 (No Color Pollution)
*   배경색 위에 반투명 오버레이를 사용할 경우, 반드시 특정 컨테이너 내부로 범위를 제한하여 전체 톤이 탁해지는 것을 방지합니다.

### 4) 시각적 위계의 절대성 (Rule of Visual Authority)
*   **정답지 (The Truth)**: `/main/corporate_light/index_ko.html` 등 **HTML 시안이 모든 디자인의 절대적 정답**입니다.
*   **도구 (The Tool)**: 본 매뉴얼의 코딩 규칙은 시안을 구현하기 위한 수단일 뿐입니다.
*   **원칙**: 매뉴얼의 규칙과 시안의 비주얼이 충돌할 경우, **무조건 시안의 비주얼(1px, 색상)을 우선**합니다.

## 2. 디자인 시스템: 토큰 및 변수 (Design System)

모든 스타일 정의 시 하드코딩(Hex/RGB 직접 입력)을 금지하며, 반드시 다음의 디자인 토큰을 참조해야 합니다.

### 1) 핵심 색상 토큰
| 변수명 | 용도 | 비고 |
| :--- | :--- | :--- |
| `--color-bg-dark` | 사이트 전체 기본 배경색 | 라이트 모드 대응 필수 |
| `--color-brand-etsy` | `#F1641E` (Etsy Orange) | 라이트 테마 푸터 및 강조색 표준 |
| `--color-accent-gold` | 브랜드 핵심 강조색 | 에센셜 포인트용 |
| `--color-text-primary` | 기본 본문 텍스트 색상 | 가독성 중심 |
| `--color-text-secondary` | 보조 텍스트 및 설명글 | 투명도 기반 설정 권장 |

### 2) 레이아웃 및 간격 토큰
| 변수명 | 기본값 | 용도 |
| :--- | :--- | :--- |
| `--spacing-container` | `1400px` | 콘텐츠 최대 가로 너비 |
| `--spacing-section` | `100px` | 섹션 간 수직 여백 |
| `--header-height` | `100px` | 상단 헤더 고정 높이 |

## 3. 타이포그래피 표준 (Typography Standards)

시각적 위계 확립을 위해 위치에 따른 **시맨틱 변수 매핑**을 준수합니다.

### 1) 섹션 타이틀 (Section Title & Edge Bar)
메인 페이지와 서브 페이지의 디자인 아이덴티티를 명확히 분리합니다.

*   **메인 페이지 (Main Page Standard)**:
    - **레이아웃**: 중앙 정렬 (`text-align: center`)
    - **에지 바**: 사용 금지. (와이드하고 개방적인 느낌 유지)
    - **변수 매핑**: `color: var(--color-title-main)`
*   **서브 페이지 (Sub Page Standard - Edge Bar)**:
    - **레이아웃**: 좌측 정렬 (`text-align: left`)
    - **에지 바**: 필수 사용. (섹션 헤더 왼쪽에 `var(--color-accent-gold)` 색상의 4px 수직선 배치)
    - **변수 매핑**: `color: var(--color-title-sub)`
*   **Hero 영역**: 배경 이미지 위의 제목은 예외적으로 **White (`#ffffff`)**를 상속받으며, 에지 바를 생략합니다.

### 2) 섹션 서브 타이틀 (Sub Title)
*   **`.section-subtitle`**: 보조 제목용 클래스.
*   **표준 속성**: `color: var(--color-accent-gold)`, `font-weight: 600`, `text-transform: uppercase`, `font-size: 0.9rem`.

## 4. 테마 및 스킨 아키텍처 (Architecture)

### 1) 허브 앤 스포크 모델 (Hub & Spoke)
*   **Hub**: `default.css` (모든 전역 변수와 공통 규격 정의)
*   **Spoke**: 플러그인/스킨 CSS (테마 변수를 상속받아 레이아웃만 구현)

### 2) 네이밍 규칙 (Namespace)
충돌 방지를 위해 접두사 사용을 권장합니다.
*   `.ci-` : Company Intro 플러그인
*   `.mc-` : Main Content Manager
*   `.theme-` : 테마 고유 컴포넌트

### 3) 디자인 스위치 (Design Switch)
메인과 서브를 구분하기 위해 `#wrapper` 클래스를 활용합니다.
```php
<div id="wrapper" <?php echo (defined('_INDEX_')) ? '' : ' class="sub-layout-light"'; ?>>
```

### 4) 서브페이지 레이아웃 (Layout Standard)
페이지의 성격에 따라 두 가지 레이아웃을 선택적으로 적용합니다.
*   **Full Width**: 와이드한 연출이 필요한 갤러리, 제품 리스트 등에 사용.
*   **Sidebar (LNB)**: 인사말, 조직도 등 뎁스 이동이 잦은 정보성 페이지에 권장.
*   **구조 표준**:
    ```html
    <div class="sub-layout-body">
        <aside class="sub-sidebar"> ...LNB Content... </aside>
        <main class="sub-content"> ...Main Content... </main>
    </div>
    ```

### 5) 스킨 위치 전략: 플러그인 vs 테마 (Skin Location Strategy)
개발 효율성과 유지보수 편의성을 위해 스킨의 성격에 따라 위치를 이원화합니다.

| 구분 | 위치 (Location) | 대상 (Target) | 선정 기준 (Why?) |
| :--- | :--- | :--- | :--- |
| **Logic-Heavy** | `/plugin/{name}/skin/` | 회사소개, 지도 API, 디자인 관리 | 복잡한 기능(PHP 로직)과 스킨이 강하게 결합된 경우. 플러그인 폴더 내에서 완결성을 가짐. |
| **Visual-Only** | `/theme/{name}/skin/` | **최신글(Latest), 게시판(Board)** | 순수한 데이터 표시 영역. 테마의 CSS(`default.css`)와 100% 일체화되어야 하므로 테마 폴더에 위치하는 것이 유리함. |

*   **Rule**: 기능성 모듈은 플러그인 폴더에 두되 `skin.head.php`로 제어하고, 순수 비주얼 위젯은 과감하게 테마 폴더(`theme/skin/...`)로 이동하여 관리합니다.

## 5. 관리자 메뉴별 상세 제작 매뉴얼 (Admin Menu Manual)

관리자 메뉴 순서(`admin.menu800.php`)에 맞춰 정리된 상세 가이드입니다.

### 1) 메인 이미지 관리 (Main Image Manager)
*   **용도**: 사이트의 첫 인상을 결정하는 히어로 영역 (슬라이드, 영상, 타이포그래피).
*   **디자인 원칙**: 고해상도 이미지와 에디토리얼 타이포그래피의 결합.
*   **핵심 변수**: 
    - `color: #ffffff` (Page Title 규칙: 이미지 위 명시성 확보)
    - `mix-blend-mode: exclusion` (프리미엄 반전 효과 권장)
*   **주의사항**: 오버레이는 반드시 `hero_bg_wrap` 내부로 제한하여 배경색 오염 방지.

### 2) 상단메뉴스킨관리 (Top Menu Manager)
*   **용도**: 사이트 전체 네비게이션(GNB).
*   **디자인 원칙**: 테마 컨셉에 맞춘 투명(Main)과 고정(Sub) 스타일의 조화.
*   **핵심 변수**: `--color-header-bg`, `--color-menu-text`.
*   **주의사항**: 스크롤 시 상태 변화(Fixed/Active)에 따른 색상 반전 로직 포함.

### 3) 메인 컨텐츠관리 (Main Content Manager)
*   **용도**: 메인 페이지를 구성하는 다양한 콘텐츠 위젯 (Story, Works, Philo 등).
*   **디자인 원칙**: 브랜드 아이덴티티 강조 및 시각적 환기.
*   **핵심 변수**: 
    - `color: var(--color-title-main)` (섹션 타이틀)
    - `color: var(--color-accent-gold)` (라인/아이콘 등 포인트)
*   **구조**: `.section-title` + `.section-subtitle` 표준 클래스 사용 필수.

### 4) 보드최근글스킨관리 (Latest Skin Manager)
*   **용도**: 메인 페이지나 섹션 내부에 최근 게시물을 노출할 때 사용하는 위젯.
*   **주요 스킨**: `Selected Works` (포트폴리오 마키), `Main News` (뉴스 리스트).
*   **디자인 원칙**: 섹션 분위기에 맞춘 컴팩트한 레이아웃과 정보의 선택적 노출.
*   **설정 표준**: 
    - **스킨**: `theme/selected_works` (비주얼 중심 위젯은 테마 폴더 사용)
    - **ID 규칙**: `corporate_main_works` ({테마}_main_works 표준)
*   **호출**: 
    ```php
    <?php latest_widget('corporate_main_works'); ?>
    ```
    또는 회사소개 에디터 내 `{LATEST_SKIN:corporate_main_works}` 치환자 사용.

### 5) 보드스킨관리 (Board Skin Manager)
*   **용도**: `/bbs/board.php`에서 구동되는 실제 게시판 (List/View/Write).
*   **디자인 원칙**: 테마 일관성을 유지하는 리스트 그리드 및 상세 페이지.
*   **핵심 변수**: 
    - `color: var(--color-title-sub)` (게시물 타이틀)
    - `border-color: var(--color-border)` (구분선)
    - `bs_theme`, `bs_lang` (DB 설정 테이블 분리)
*   **구조**: `skin.head.php`를 활용한 테마 변수 주입 및 CSS 일원화.

### 6) 서브디자인관리 (Sub Design)
*   **용도**: 서브 페이지 상단 메뉴별 맞춤형 비주얼 영역.
*   **디자인 원칙**: 페이지 전환 시 시각적 안정감 및 위계 제공.
*   **핵심 변수**: `color: #ffffff` (배경 이미지 위 화이트 상속).
*   **구조**: 메뉴 코드 기반 자동 매칭 이미지 호출 로직 준수.
*   **프리미엄 스킨: Instinct (Editorial)**
    - **레이아웃**: 500px 고정 이미지(왼쪽) + 텍스트(오른쪽)의 **비대칭 그리드**.
    - **타이포그래피**: `clamp(60px, 9vw, 130px)`, `line-height: 0.9` (초대형/초밀착).
    - **Zero-Gap Point**: 브레드크럼과 `WHO WE ARE` 태그의 정밀한 여백 배치 준수.

### 7) 회사소개 관리 (Company Intro)
*   **용도**: 인사말, 연혁, 조직도 등 정형화된 기업 정보 시각화.
*   **디자인 원칙**: 에디토리얼 가독성과 신뢰감 있는 레이아웃.
*   **핵심 변수**: 
    - `color: var(--color-title-sub)` (정보 전달용 가독성 컬러)
    - `line-height: 1.8` (본문 가독성 확보)
*   **구조**: `.ci-skin-wrapper` 래퍼를 통한 상하 여백 및 중앙 정렬 통일.

### 8) 온라인 문의 관리 (Online Inquiry)
*   **용도**: 온라인 문의, 견적 요청 등 사용자 입력 폼.
*   **디자인 원칙**: 깔끔한 입력창 디자인과 명확한 CTA 버튼.
*   **핵심 변수**: 
    - `--mc-btn-primary` (강조형 버튼)
    - `background: var(--color-bg-panel)` (폼 배경)
*   **주의사항**: 모든 버튼은 `.mc-btn` 공통 시스템을 상속받아 제작.

### 9) 카피라이트관리 (Copyright Manager)
*   **용도**: 기업 정보, 하단 로고, SNS 링크 공유 영역.
*   **디자인 원칙**: 정보 전달의 명확성과 테마 톤앤매너 마무리.
*   **핵심 스킨**: **Style D (Editorial Bold)** - 엣시(Etsy) 오렌지 배경과 초대형 타이포그래피.
*   **디자인 규격 (Source: `index_ko.html`)**:
    - **Mega Title**: `15vw` (반드시 `vw` 사용), `line-height: 0.8`, `letter-spacing: -5px`.
    - **Logo Layout**: 로고는 반드시 좌측 상단 (`fs-top`)에 위치하며, **슬로건 바로 위에 중복 노출되지 않아야 함**. (시안 대비 불필요한 요소 제거).
    - **Padding**: `10vh 4vw 5vh` (대담한 에디토리얼 여백).
    - **Copyright**: 시안에 없는 경우 과감히 생략하거나 최하단에 아주 작게 배치하여 메가 슬로건의 임팩트를 방해하지 말 것.
*   **배경색 원칙 (Semantic First)**: 
    - 스킨에 직접 브랜드 색상을 적는 것은 '디자인 권력 남용'입니다.
    - **표준 매핑**: 테마(`default.css`)에서 `--color-bg-footer`를 정의하고, 스킨은 오직 이 변수(또는 `--cp-bg`)만 상속받습니다.
    - **Default Value**: 라이트 테마 시안 기준 `--color-bg-footer: var(--color-brand);` (#FF3B30).
*   **다국어 및 ID 표준 (Pattern A)**:
    - **자동 ID**: 테마+언어 선택 시 자동 생성. 한국어는 접미사를 생략하는 것이 표준입니다.
    - **No Hardcoding**: 'CONTACT', 'ADDRESS' 등의 라벨은 반드시 `{tel_name}`, `{addr_name}` 치환자로 처리하여 관리자에서 다국어 대응이 가능해야 합니다.
*   **재발 방지 (Zero-Gap Adjustment)**:
    - 시안(`index_ko.html`)과 비교하여 슬로건이 작아 보인다면, 컨테이너의 `max-width`가 제한되어 있는지 확인하고 이를 해제하십시오. 에디토리얼 디자인은 '풀 와이드'의 개방감이 핵심입니다.


### 10) 지도 API 설정 (Map API)
*   **용도**: 구글/카카오/네이버 지도 연동 설정.
*   **원칙**: API 키는 DB에서 관리하며, 스킨에서는 `{MAP_API_KEY}` 치환자 사용.

### 11) 트리 카테고리 관리 (Tree Category)
*   **용도**: 계층형 분류 관리 (쇼핑몰, 포트폴리오 분류 등).
*   **원칙**: 무한 뎁스 지원 및 Drag & Drop UI.

## 6. 생산성 극대화 도구 (Production Tools)

## 7. 스킨 제작 사이드 인프라 (Production Workflow)

디자인 시안 확정 후 실제 코드로 구현하는 '스킨 제작 사이드'의 표준 작업 프로세스입니다.

### 1) 금형(Structure) 제작
스킨 제작 시 특정 색상을 고집하는 대신, 테마에서 공급될 변수가 담길 '틀'을 먼저 만듭니다.
```css
.section-title {
    color: var(--color-title-main); /* 메인용 금형 */
    font-family: var(--font-heading);
}
```

### 2) 시안 변수 주입
디자인 시안에서 결정된 Hex 값은 오직 테마의 `default.css`에만 입력합니다. 이로써 테마만 교체하면 모든 스킨의 색상이 한 번에 변경됩니다.

### 3) 컴포넌트 중심 제작 방식 (Component-based Workflow)
'회사소개(Company Intro)' 플러그인을 단순한 페이지가 아닌, 사이트 곳곳에서 재사용할 수 있는 **'디자인 컴포넌트 저장소'**로 활용합니다.
*   **제작 단계**:
    1.  특정 목적(예: 메인지도, 연혁)에 맞는 전용 스킨을 제작합니다.
    2.  스킨 헤더에 데이터 바인딩(`{CP_ADDRESS}` 등)과 에디토리얼 타이포그래피를 미리 구성합니다.
    3.  완성된 컨텐츠(컴포넌트)를 '메인 컨텐츠 관리자'나 '서브 디자인 관리자'에서 호출하여 배치합니다.
*   **효율성 포인트**: 컨텐츠 수정이 필요할 때 오직 회사소개 관리자 한 곳만 수정하면, 이를 호출한 메인/서브의 모든 영역이 동시에 최신화됩니다.

### 4) 섹션 간격 가이드 (Spacing Protocol)
메인 페이지에서 여러 섹션이 결합될 때 발생하는 '간격 좁음' 현상을 방지하기 위한 표준 지침입니다.
*   **원칙**: 모든 섹션은 상하단에 최소 100px(`--spacing-section`) 이상의 여백을 확보해야 합니다.
*   **에디토리얼 여백**: 비주얼 임팩트가 중요한 섹션(메인지도, 히어로 등)은 `12vh` ~ `15vh` 수준의 더 넓은 여백을 대담하게 사용하십시오.
*   **충돌 방지**: 
    - 인접한 두 섹션의 배경색이 동일할 경우 여백이 합쳐져 너무 넓어 보일 수 있습니다.
    - 배경색이 다를 경우 각 섹션의 내부 패딩이 부족하면 컨텐츠가 배경색 경계에 너무 딱 붙어 답답해 보입니다.
*   **해결**: 모든 스킨은 최상위 래퍼에 `padding: var(--spacing-section, 12vh) 0;`과 같은 변수 기반 패딩을 강제 적용하여 테마 차원에서 통제 가능하게 합니다.

## 8. 에디터 환경 안정화 (Editor Stabilization)

스마트에디터(SmartEditor2)와 같은 Iframe 기반 에디터를 사용할 때는 브라우저 로딩 시점과 에디터 내부의 스타일 리셋을 제어하는 '안정화 로직'이 필수적입니다.

### 1) 테마 주권 강제 상속 (Theme Style Injection)
에디터 내부(`iframe`)는 부모 페이지의 CSS를 인식하지 못합니다. 따라서 자바스크립트를 통해 테마의 CSS 변수가 담긴 `default.css`를 직접 주입해야 합니다.
*   **핵심 로직**: 에디터 `body`에 `!important`를 활용한 스타일 규칙 주입.
*   **JS Fallback**: CSS 변수가 로드되기 전의 찰나의 순간(흰색 번쩍임)을 방지하기 위해 `doc.body.style.backgroundColor`를 즉시 설정합니다.

### 2) 동적 초기화 및 감시 루프 (Initialization & Enforcement)
SmartEditor2는 초기화 과정에서 내부 프레임을 여러 번 다시 그리며 스타일을 초기화할 수 있습니다. 
*   **Global Init**: 에디터 초기화 코드(`get_editor_js`)는 반드시 함수 외부(전역)에서 실행되어 `oEditors` 객체를 미리 생성해야 합니다.
*   **Ready Check**: `setTimeout`을 통해 에디터 객체가 완벽히 바인딩 되었는지 확인 후 로직을 가동합니다.
*   **Enforcement Interval**: 로딩 직후 약 5초간 주기적인(예: 0.5초 간격) 다시 주입 루프를 실행하여 에디터의 자가 리셋 시도를 무력화합니다.

### 3) 관리자 편집 환경 최적화 (Admin UX Optimization)
프리미엄 에디토리얼 스킨은 일반적인 게시물보다 수직 길이가 매우 깁니다. 기본 에디터 높이(300px)에서는 전체 레이아웃 확인이 어려우므로 관리자 페이지 자바스크립트를 통한 높이 최적화가 필수입니다.
*   **표준 높이**: 프리미엄 시안 작업 시 에디터 높이를 최소 `700px` ~ `1000px` 이상으로 강제 설정합니다.
*   **JS 리사이징 로직**: `forceResizeEditor()`와 같은 함수를 초기화 시퀀스에 포함하여 에디터 iframe의 높이를 디자인 규모에 맞게 조정하십시오.

### 4) 글로벌 스타일 상속 차단 (Breaking Inheritance)
테마의 전역 스타일(`style.css`의 `.section-header` 등)이 개별 스킨의 디자인을 오염시키는 경우가 빈번합니다. 특히 프리미엄 디자인 시안은 독자적인 레이아웃을 요구하므로, 전역 클래스 사용을 지양해야 합니다.
*   **Unique Class Naming**: `.section-header`와 같은 공통 클래스 대신 `.inquiry-header-premium`처럼 스킨 고유의 클래스명을 사용하여 전역 스타일(`border-left` 등)의 간섭을 원천 차단하십시오.
*   **Explicit Override**: 부득이하게 공통 클래스를 사용해야 한다면, 스킨 CSS 최상단에서 `border: none !important; padding: 0 !important;`와 같이 초기화 스타일을 명시적으로 선언해야 합니다.

## 10. 스킨 관리자 최적화 및 이슈 대응 (Optimization & Troubleshooting)
**[2026.01 Update]** 최근글(Latest) 및 게시판(Board) 스킨 관리자 구축 과정에서 도출된 효율성 극대화 매뉴얼입니다.

### 1) Wrapper Skin 전략 (Editor vs Frontend)
에디터 화면과 실제 출력 화면이 다른 것은 **정상**이며, 이는 의도된 설계입니다.
*   **Wrapper Skin (`main_latest.html`)**: 에디터에서는 `{LATEST_SKIN_DISPLAY}`와 같은 치환자만 보여줍니다.
*   **작동 원리**: 
    1.  사용자가 에디터에서 스킨을 선택하고 저장.
    2.  `view.php`가 해당 스킨을 로드.
    3.  스킨 내부의 치환자(`{...}`)를 감지하여 실제 위젯(PHP 함수)으로 실시간 교체.
*   **장점**: 에디터 내용을 일일이 수정하지 않아도, 위젯 설정만 바꾸면 디자인이 즉시 변경됨.

### 2) ID 매칭 자동화 (Pattern B의 핵심)
위젯 연결 실패의 90%는 **ID 불일치**에서 발생합니다. 이를 방지하기 위해 'Pattern B'를 도입했습니다.
*   **AS-IS (기존)**: 사용자가 ID를 직접 입력 (오타 발생 가능성 높음).
*   **TO-BE (개선)**: **테마(Theme)**와 **언어(Lang)**를 선택하면 `ls_id`가 자동으로 조합됨.
    *   예: `corporate` + `ko` -> `corporate_main_works`
*   **주의사항**: DB 레거시 데이터(직접 입력된 숫자 ID 등)가 남아있을 경우, 연결이 안 될 수 있습니다. **반드시 DB의 `ls_id` 컬럼을 확인**하십시오.

### 3) 디버깅 속성 코스 (Troubleshooting Checklist)
위젯이 화면에 안 나올 때 다음 순서대로 체크하면 1분 안에 해결됩니다.
1.  **ID 확인**: DB(`g5_plugin_latest_skin_config`)를 열어 실제 `ls_id`가 무엇인지 확인 (예: 숫자 `4`인지, 문자열 `corporate`인지).
2.  **함수 인자 확인**: `latest_widget('ID')` 호출 시, 정확한 ID를 넘기고 있는지 확인.
3.  **타입 에러(TypeError)**: `latest.lib.php`의 인자 순서(5번째 vs 6번째) 이슈인지 확인 (최신 버전은 `$options` 배열을 6번째에 넘겨야 함).

## 11. 5초 컷 스타터 킷 (Instant Starter Kit)
**고민하지 말고 복사해서 쓰십시오. 스킨 제작 시간을 단축하는 핵심 자산입니다.**

### A. 표준 PHP 템플릿 (`skin.head.php` 포함)
```php
<?php
if (!defined('_GNUBOARD_')) exit;
// [Standard] 공통 헤더 로드 (경로/설정/헤더 처리)
include_once(dirname(__FILE__) . '/../skin.head.php');
$uniqid = uniqid('skin_');
?>

<!-- [Wrapper] 테마 변수 상속 및 래퍼 -->
<section class="mc-skin-wrapper <?php echo $skin_cls; ?>" id="<?php echo $uniqid; ?>">
    <style>
        /* [Scoped CSS] 오직 이 스킨 내부만 영향 */
        #<?php echo $uniqid; ?> .section-title { color: var(--color-title-main); }
        #<?php echo $uniqid; ?> .mc-btn { margin-top: 30px; }
    </style>

    <div class="container">
        <!-- [Header] 표준 타이틀 영역 -->
        <div class="section-header text-center">
            <h2 class="section-title" data-aos="fade-up">
                <?php echo $txt_title; // 다국어 변수 사용 ?>
            </h2>
            <?php if ($txt_desc) { ?>
            <p class="section-subtitle" data-aos="fade-up" data-aos-delay="100">
                <?php echo $txt_desc; ?>
            </p>
            <?php } ?>
        </div>

        <!-- [Body] 콘텐츠 영역 -->
        <div class="skin-content">
            <!-- Your Code Here -->
        </div>
    </div>
</section>
```

### B. 표준 DB 스키마 패턴 (Standard DB Schema)
테이블 생성 시 반드시 다음 컬럼을 포함하여 **테마/언어 확장성**을 보장해야 합니다.
```sql
`mo_theme` varchar(50) NOT NULL DEFAULT 'corporate', -- 테마 구분
`mo_lang` varchar(10) NOT NULL DEFAULT 'ko',         -- 언어 구분 (ko/en)
`mo_sort` int(11) NOT NULL DEFAULT '0',              -- 정렬 순서
`mo_active` tinyint(4) NOT NULL DEFAULT '1'          -- 노출 여부
```

### C. 다국어 텍스트 처리 규칙 (Multi-Language Rules)
스킨 파일 내에 한글 텍스트 하드코딩을 절대 금지합니다.
*   **[BAD]** `<h2>주요 실적</h2>`
*   **[GOOD]** `<h2><?php echo $txt_title; ?></h2>`
*   **구현:** `view.php` 또는 `skin.head.php`에서 언어(`$lang`)에 따라 변수를 정의해서 넘겨줍니다.

---

### D. 플러그인 UI 표준 - Pattern B (NEW)
신규 플러그인 또는 기존 플러그인 배포 시 다음 UI/UX 표준을 반드시 준수합니다.

#### 1. 식별코드(ID) 생성 규칙
- **구조:** `{테마}_{언어}_{커스텀명}`
- **한국어 규칙:** 한국어(기본)의 경우 관리 편의 및 시각적 간결함을 위해 언어 코드(`kr`, `ko`)를 **생략**합니다.
  - GOOD: `corporate`, `corporate_sub`
  - BAD: `corporate_kr`, `corporate_ko`
- **다국어 규칙:** 영어(`en`), 일본어(`jp`), 중국어(`cn`) 등은 반드시 접미사를 붙입니다.
- **입력 제한:** 커스텀 이름 입력란에는 반드시 JavaScript `replace(/[^a-z0-9_]/gi, '')` 필터를 적용하여 한글 및 특수문자 입력을 원천 차단합니다.

#### 2. 셀렉션 UI 구성
- **배치:** 테마 선택과 언어 선택 드롭다운은 `display:flex`를 사용하여 가로로 배치합니다.
- **초기값:** 드롭다운의 초기값은 반드시 "테마 선택", "언어 선택" 등 비어있는 상태로 두어, 관리자가 명확히 선택했을 때만 ID가 생성되도록 합니다.

#### 3. 상태 피드백 (State Feedback)
- 생성된 ID를 사용자가 즉시 확인할 수 있도록 별도의 시각적 영역(예: `<strong>` 태그)에 실시간으로 표시합니다.

## 13. 재발 방지 및 품질 유지 수칙 (Prevention & Quality Enforcement)

테마 제작 시 반복되는 실수를 방지하고, 어떤 작업자가 투입되어도 동일한 고품질을 유지하기 위한 **'절대 금기'** 및 **'필수 체크'** 리스트입니다.

### 1) 변수 우선순위 (Variable Hierarchy)
코딩 전 반드시 다음의 3단계 변수 매핑을 설계하십시오.
1.  **Primitive**: `#F1641E` (단순 Hex값)
2.  **Brand Token**: `--color-brand-etsy` (브랜드 자산)
