# 테마스킨 제작 가이드 (Theme Skin Development Guide)

이 문서는 그누보드 테마 및 플러그인 스킨의 일관성을 유지하고, 프리미엄한 디자인 품질을 보장하기 위한 공식 제작 표준을 정의합니다. 모든 제작자는 본 가이드를 준수하여 '테마 주권'이 확립된 환경에서 스킨을 제작해야 합니다.

---

## 1. 제작 철학 및 핵심 원칙 (Core Principles)

### 1) 테마 주권 (Theme Sovereignty)
플러그인이나 개별 스킨은 '구조(Structure)'만 결정하며, 모든 '시각적 속성(Property: 색상, 폰트, 여백)'은 테마의 `default.css`가 결정합니다.

### 2) 비움의 미학 (Negative Space)
프리미엄 디자인의 핵심은 '비움'에 있습니다. 과감한 마진(Margin)과 패딩(Padding)을 사용하여 타이포그래피가 숨을 쉴 수 있는 공간을 확보하십시오.

### 3) 색상 오염 방지 (No Color Pollution)
배경색 위에 반투명 오버레이를 사용할 경우, 반드시 특정 컨테이너 내부로 범위를 제한하여 전체 톤이 탁해지는 것을 방지합니다.

---

## 2. 디자인 시스템: 토큰 및 변수 (Design System)

모든 스타일 정의 시 하드코딩(Hex/RGB 직접 입력)을 금지하며, 반드시 다음의 디자인 토큰을 참조해야 합니다.

### 1) 핵심 색상 토큰
| 변수명 | 용도 | 비고 |
| :--- | :--- | :--- |
| `--color-bg-dark` | 사이트 전체 기본 배경색 | 라이트 모드 대응 필수 |
| `--color-accent-gold` | 브랜드 핵심 강조색 | 에센셜 포인트용 |
| `--color-text-primary` | 기본 본문 텍스트 색상 | 가독성 중심 |
| `--color-text-secondary` | 보조 텍스트 및 설명글 | 투명도 기반 설정 권장 |

### 2) 레이아웃 및 간격 토큰
| 변수명 | 기본값 | 용도 |
| :--- | :--- | :--- |
| `--spacing-container` | `1400px` | 콘텐츠 최대 가로 너비 |
| `--spacing-section` | `100px` | 섹션 간 수직 여백 |
| `--header-height` | `100px` | 상단 헤더 고정 높이 |

---

## 3. 타이포그래피 표준 (Typography Standards)

시각적 위계 확립을 위해 위치에 따른 **시맨틱 변수 매핑**을 준수합니다.

### 1) 섹션 타이틀 (Section Title)
메인과 서브의 독립적인 디자인 제어를 위해 두 개의 변수를 사용합니다.
*   **`.section-title`**: 모든 섹션의 메인 제목용 클래스.
*   **변수 매핑**: 
    - 메인 페이지 위젯: `color: var(--color-title-main)`
    - 서브 페이지 컨텐츠: `color: var(--color-title-sub)`
*   **Hero 영역**: 배경 이미지 위의 제목은 예외적으로 **White (`#ffffff`)**를 상속받습니다.

### 2) 섹션 서브 타이틀 (Sub Title)
*   **`.section-subtitle`**: 보조 제목용 클래스.
*   **표준 속성**: `color: var(--color-text-secondary)`, `font-size: 1.1rem`, `letter-spacing: 0.1em`.

## 4. 테마 및 스킨 아키텍처 (Architecture)

### 1) 허브 앤 스포크 모델 (Hub & Spoke)
*   **Hub**: `default.css` (모든 전역 변수와 공통 규격 정의)
*   **Spoke**: 플러그인/스킨 CSS (테마 변수를 상속받아 레이아웃만 구현)

### 2) 네이밍 규칙 (Namespace)
충돌 방지를 위해 접두사 사용을 권장합니다.
*   `.ci-` : Company Intro 플러그인
*   `.mc-` : Main Content Manager
*   `.theme-` : 테마 고유 컴포넌트

### 3) 디자인 스위치 (Design Switch)
메인과 서브를 구분하기 위해 `#wrapper` 클래스를 활용합니다.
```php
<div id="wrapper" <?php echo (defined('_INDEX_')) ? '' : ' class="sub-layout-light"'; ?>>
```

## 5. 8대 스킨 유형별 상세 제작 매뉴얼 (Skin Type Manual)

시안과 구현 사이의 오차를 줄이고 작업 속도를 극대화하기 위한 유형별 상세 가이드입니다.

### 1) 메인 비주얼형 (Main Image Manager)
*   **용도**: 사이트의 첫 인상을 결정하는 히어로 영역 (슬라이드, 영상, 타이포그래피).
*   **디자인 원칙**: 고해상도 이미지와 에디토리얼 타이포그래피의 결합.
*   **핵심 변수**: 
    - `color: #ffffff` (Page Title 규칙: 이미지 위 명시성 확보)
    - `mix-blend-mode: exclusion` (프리미엄 반전 효과 권장)
*   **주의사항**: 오버레이는 반드시 `hero_bg_wrap` 내부로 제한하여 배경색 오염 방지.

### 2) 상단 메뉴형 (Top Menu Manager)
*   **용도**: 사이트 전체 네비게이션(GNB).
*   **디자인 원칙**: 테마 컨셉에 맞춘 투명(Main)과 고정(Sub) 스타일의 조화.
*   **핵심 변수**: `--color-header-bg`, `--color-menu-text`.
*   **주의사항**: 스크롤 시 상태 변화(Fixed/Active)에 따른 색상 반전 로직 포함.

### 3) 메인 섹션형 (Main Content Manager)
*   **용도**: 메인 페이지를 구성하는 다양한 콘텐츠 위젯 (Story, Works, Philo 등).
*   **디자인 원칙**: 브랜드 아이덴티티 강조 및 시각적 환기.
*   **핵심 변수**: 
    - `color: var(--color-title-main)` (섹션 타이틀)
    - `color: var(--color-accent-gold)` (라인/아이콘 등 포인트)
*   **구조**: `.section-title` + `.section-subtitle` 표준 클래스 사용 필수.

### 4) 회사소개형 (Company Intro)
*   **용도**: 인사말, 연혁, 조직도 등 정형화된 기업 정보 시각화.
*   **디자인 원칙**: 에디토리얼 가독성과 신뢰감 있는 레이아웃.
*   **핵심 변수**: 
    - `color: var(--color-title-sub)` (정보 전달용 가독성 컬러)
    - `line-height: 1.8` (본문 가독성 확보)
*   **구조**: `.ci-skin-wrapper` 래퍼를 통한 상하 여백 및 중앙 정렬 통일.

### 5) 서브 히어로형 (Sub Design)
*   **용도**: 서브 페이지 상단 메뉴별 맞춤형 비주얼 영역.
*   **디자인 원칙**: 페이지 전환 시 시각적 안정감 및 위계 제공.
*   **핵심 변수**: `color: #ffffff` (배경 이미지 위 화이트 상속).
*   **구조**: 메뉴 코드 기반 자동 매칭 이미지 호출 로직 준수.

### 6) 게시판 스킨형 (Board Skin Manager)
*   **용도**: 일반 게시판, 포트폴리오(Works), 갤러리 리스트 및 뷰.
*   **디자인 원칙**: 테마 일관성을 유지하는 리스트 그리드 및 상세 페이지.
*   **핵심 변수**: 
    - `color: var(--color-title-sub)` (게시물 타이틀)
    - `border-color: var(--color-border)` (구분선)
*   **구조**: `skin.head.php`를 활용한 테마 변수 주입 및 CSS 일원화.

### 7) 비즈니스 폼형 (Contact Form / Online Inquiry)
*   **용도**: 온라인 문의, 견적 요청 등 사용자 입력 폼.
*   **디자인 원칙**: 깔끔한 입력창 디자인과 명확한 CTA 버튼.
*   **핵심 변수**: 
    - `--mc-btn-primary` (강조형 버튼)
    - `background: var(--color-bg-panel)` (폼 배경)
*   **주의사항**: 모든 버튼은 `.mc-btn` 공통 시스템을 상속받아 제작.

### 8) 하단 푸터형 (Copyright Manager)
*   **용도**: 기업 정보, 하단 로고, SNS 링크 공유 영역.
*   **디자인 원칙**: 정보 전달의 명확성과 테마 톤앤매너 마무리.
*   **핵심 변수**: `color: var(--text-muted)` (보조 텍스트).
*   **구조**: `{tel}`, `{email}` 등 표준 치환자를 사용한 데이터 바인딩 필수.

---

### 9) 공통 데이터 바인딩 (Data Binding)
*   **용도**: 기업 기본 정보(주소, 연락처 등)를 스킨에 자동으로 주입.
*   **표준 치환자 (Copyright Manager 연동)**:
    - `{CP_ADDRESS}`: 회사 주소
    - `{CP_TEL}`: 대표 전화번호
    - `{CP_FAX}`: 팩스 번호
    - `{CP_EMAIL}`: 대표 이메일
*   **디자인 가이드**: 정보 나열 시 `<strong>` 태그로 라벨을 감싸고, 테마의 강조색(`var(--color-accent-gold)`)을 적용하여 시각적 위계를 확보합니다.

## 6. 생산성 극대화 도구 (Production Tools)

디자인 시안을 코드로 옮기는 속도를 2배 이상 높여주는 실전 도구 모음입니다.

### 1) 타이포그래피 시맨틱 변수 치트시트
| 대상 | CSS 속성 | 사용하는 변수 | 비고 |
| :--- | :--- | :--- | :--- |
| **메인 섹션 제목** | `color` | `var(--color-title-main)` | 브랜드 강조색 연동 |
| **서브 페이지 제목** | `color` | `var(--color-title-sub)` | 가독성용 기본색 연동 |
| **히어로/비주얼 제목** | `color` | `#ffffff` | 이미지 위 고정 화이트 |
| **본문/설명 텍스트** | `color` | `var(--color-text-secondary)` | 부드러운 가독성 |
| **강조 포인트/라인** | `background` | `var(--color-accent-gold)` | 시각적 환기 |

### 2) 통합 보일러플레이트 (Starter Kit)
새 스킨 제작 시 다음 구조를 복사하여 시작하십시오.

```html
<style>
    /* [Infrastructure] 테마 변수 상속 */
    .standard-skin-wrap {
        width: 100%;
        max-width: var(--spacing-container, 1400px);
        margin: 0 auto;
        padding: var(--spacing-section, 100px) 20px;
    }
    
    .standard-skin-wrap .section-header { margin-bottom: 60px; text-align: center; }
    
    .standard-skin-wrap .section-title {
        color: var(--color-title-main); /* 혹은 title-sub */
        font-family: var(--font-heading);
        font-size: 2.8rem;
        font-weight: 800;
        text-transform: uppercase;
    }
</style>

<section class="standard-skin-wrap">
    <div class="section-header">
        <h2 class="section-title">YOUR DESIGN TITLE</h2>
        <p class="section-subtitle">Your subtitle goes here</p>
    </div>
    <div class="design-body">
        <!-- 여기서부터 디자인 시안 구현 -->
    </div>
</section>
```

## 7. 스킨 제작 사이드 인프라 (Production Workflow)

디자인 시안 확정 후 실제 코드로 구현하는 '스킨 제작 사이드'의 표준 작업 프로세스입니다.

### 1) 금형(Structure) 제작
스킨 제작 시 특정 색상을 고집하는 대신, 테마에서 공급될 변수가 담길 '틀'을 먼저 만듭니다.
```css
.section-title {
    color: var(--color-title-main); /* 메인용 금형 */
    font-family: var(--font-heading);
}
```

### 2) 시안 변수 주입
디자인 시안에서 결정된 Hex 값은 오직 테마의 `default.css`에만 입력합니다. 이로써 테마만 교체하면 모든 스킨의 색상이 한 번에 변경됩니다.

### 3) 컴포넌트 중심 제작 방식 (Component-based Workflow)
'회사소개(Company Intro)' 플러그인을 단순한 페이지가 아닌, 사이트 곳곳에서 재사용할 수 있는 **'디자인 컴포넌트 저장소'**로 활용합니다.
*   **제작 단계**:
    1.  특정 목적(예: 메인지도, 연혁)에 맞는 전용 스킨을 제작합니다.
    2.  스킨 헤더에 데이터 바인딩(`{CP_ADDRESS}` 등)과 에디토리얼 타이포그래피를 미리 구성합니다.
    3.  완성된 컨텐츠(컴포넌트)를 '메인 컨텐츠 관리자'나 '서브 디자인 관리자'에서 호출하여 배치합니다.
*   **효율성 포인트**: 컨텐츠 수정이 필요할 때 오직 회사소개 관리자 한 곳만 수정하면, 이를 호출한 메인/서브의 모든 영역이 동시에 최신화됩니다.

### 4) 섹션 간격 가이드 (Spacing Protocol)
메인 페이지에서 여러 섹션이 결합될 때 발생하는 '간격 좁음' 현상을 방지하기 위한 표준 지침입니다.
*   **원칙**: 모든 섹션은 상하단에 최소 100px(`--spacing-section`) 이상의 여백을 확보해야 합니다.
*   **에디토리얼 여백**: 비주얼 임팩트가 중요한 섹션(메인지도, 히어로 등)은 `12vh` ~ `15vh` 수준의 더 넓은 여백을 대담하게 사용하십시오.
*   **충돌 방지**: 
    - 인접한 두 섹션의 배경색이 동일할 경우 여백이 합쳐져 너무 넓어 보일 수 있습니다.
    - 배경색이 다를 경우 각 섹션의 내부 패딩이 부족하면 컨텐츠가 배경색 경계에 너무 딱 붙어 답답해 보입니다.
*   **해결**: 모든 스킨은 최상위 래퍼에 `padding: var(--spacing-section, 12vh) 0;`과 같은 변수 기반 패딩을 강제 적용하여 테마 차원에서 통제 가능하게 합니다.

## 8. 에디터 환경 안정화 (Editor Stabilization)

스마트에디터(SmartEditor2)와 같은 Iframe 기반 에디터를 사용할 때는 브라우저 로딩 시점과 에디터 내부의 스타일 리셋을 제어하는 '안정화 로직'이 필수적입니다.

### 1) 테마 주권 강제 상속 (Theme Style Injection)
에디터 내부(`iframe`)는 부모 페이지의 CSS를 인식하지 못합니다. 따라서 자바스크립트를 통해 테마의 CSS 변수가 담긴 `default.css`를 직접 주입해야 합니다.
*   **핵심 로직**: 에디터 `body`에 `!important`를 활용한 스타일 규칙 주입.
*   **JS Fallback**: CSS 변수가 로드되기 전의 찰나의 순간(흰색 번쩍임)을 방지하기 위해 `doc.body.style.backgroundColor`를 즉시 설정합니다.

### 2) 동적 초기화 및 감시 루프 (Initialization & Enforcement)
SmartEditor2는 초기화 과정에서 내부 프레임을 여러 번 다시 그리며 스타일을 초기화할 수 있습니다. 
*   **Global Init**: 에디터 초기화 코드(`get_editor_js`)는 반드시 함수 외부(전역)에서 실행되어 `oEditors` 객체를 미리 생성해야 합니다.
*   **Ready Check**: `setTimeout`을 통해 에디터 객체가 완벽히 바인딩 되었는지 확인 후 로직을 가동합니다.
*   **Enforcement Interval**: 로딩 직후 약 5초간 주기적인(예: 0.5초 간격) 다시 주입 루프를 실행하여 에디터의 자가 리셋 시도를 무력화합니다.

### 3) 관리자 편집 환경 최적화 (Admin UX Optimization)
프리미엄 에디토리얼 스킨은 일반적인 게시물보다 수직 길이가 매우 깁니다. 기본 에디터 높이(300px)에서는 전체 레이아웃 확인이 어려우므로 관리자 페이지 자바스크립트를 통한 높이 최적화가 필수입니다.
*   **표준 높이**: 프리미엄 시안 작업 시 에디터 높이를 최소 `700px` ~ `1000px` 이상으로 강제 설정합니다.
*   **JS 리사이징 로직**: `forceResizeEditor()`와 같은 함수를 초기화 시퀀스에 포함하여 에디터 iframe의 높이를 디자인 규모에 맞게 조정하십시오.

### 4) 글로벌 스타일 상속 차단 (Breaking Inheritance)
테마의 전역 스타일(`style.css`의 `.section-header` 등)이 개별 스킨의 디자인을 오염시키는 경우가 빈번합니다. 특히 프리미엄 디자인 시안은 독자적인 레이아웃을 요구하므로, 전역 클래스 사용을 지양해야 합니다.
*   **Unique Class Naming**: `.section-header`와 같은 공통 클래스 대신 `.inquiry-header-premium`처럼 스킨 고유의 클래스명을 사용하여 전역 스타일(`border-left` 등)의 간섭을 원천 차단하십시오.
*   **Explicit Override**: 부득이하게 공통 클래스를 사용해야 한다면, 스킨 CSS 최상단에서 `border: none !important; padding: 0 !important;`와 같이 초기화 스타일을 명시적으로 선언해야 합니다.

## 9. 품질 검증 체크리스트 (Checklist)

*   **[ ] Variable Inheritance**: 모든 폰트와 간격이 테마 변수를 상속받는가?
*   **[ ] Semantic Alignment**: 메인/서브 타이틀 변수가 용도에 맞게 분리되었는가?
*   **[ ] Mobile Readiness**: `mobile.css` 혹은 미디어 쿼리를 통해 반응형 구조가 잡혔는가?
*   **[ ] Editor Stabilization**: 다크 테마 에디터 사용 시 배경색 번쩍임 및 리셋 방지 로직이 적용되었는가?
