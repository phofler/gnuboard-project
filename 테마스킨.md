# 테마 디자인 및 CSS 규격 가이드 (Theme Design & CSS Standards)

이 문서는 앞으로 제작할 그누보드 테마의 일관성을 유지하고, 작업 효율을 극대화하기 위한 디자인 시스템 및 CSS 작성 규칙을 정의합니다.

## 1.# 그누보드 테마 & 스킨 가이드 (`theme_skin.md`)

## 1. 테마 전략 (Theme Strategy)
현재 개발 중인 테마는 **Corporate Dark (v1.0)** 입니다.
모든 디자인은 **Dark Theme**를 기본으로 하며, 추후 **Light Theme**로의 '빠르고 쉬운 전환'을 목표로 CSS 변수(Variables)를 체계적으로 관리합니다.

### 🌓 Dark vs Light 버전 관리 규칙
1.  **Master Dark**: 현재 작업 중인 다크 버전이 마스터입니다. 모든 구현은 다크 모드 기준으로 완성합니다.
2.  **Variable-based Switching**: 색상 값은 절대 하드코딩하지 않고 반드시 `:root` 변수를 사용합니다.
    *   ❌ `color: #ffffff;` (하드코딩 금지)
    *   ✅ `color: var(--text-color);` (변수 사용)
3.  **Light Version 계획**: `default.css`의 `:root` 섹션만 변경하면 라이트 버전이 되도록 설계합니다.

## 2. 테마 CSS 정의의 5가지 핵심 요소
테마 파일(`default.css`)에는 **"모든 페이지 공통"**이거나 **"브랜드 아이덴티티"**와 직결된 5가지만 정의합니다.

### 1) 디자인 토큰 (Design Tokens) - `변수 정의`
가장 중요합니다. 색상, 폰트, 여백 등을 변수(`:root`)로 정의합니다.
*   **Colors**: `--primary-color`, `--accent-color`, `--bg-dark`, `--text-main`
*   **Fonts**: `--font-heading` (제목용), `--font-body` (본문용)
*   **Spacing**: `--container-width` (1200px 등), `--section-padding` (100px 등)

### 2) 타이포그래피 (Typography) - `글꼴 규칙`
`<h1>` ~ `<h6>`, `<p>`, `<a>` 태그의 기본 생김새입니다.
*   "모든 제목은 굵은 고딕체 (`Noto Sans KR` Bold)를 쓴다."
*   "모든 본문 줄간격(`line-height`)은 1.6이다."
*   "모든 링크(`a`)는 hover시 밑줄이 사라지고 색이 변한다."

### 3) 레이아웃 뼈대 (Layout Structure) - `공통 구조`
페이지라면 무조건 존재하는 영역들의 위치입니다.
*   **Header (#hd)**: 로고, GNB 메뉴의 위치와 배경.
*   **Wrapper & Container**: 전체 내용을 감싸는 폭과 중앙 정렬.
*   **Sub Visual (Hero)**: "회사소개", "제품소개" 같은 상단 큰 이미지 영역의 **높이(height)**와 **제목 위치**. (페이지 간 이동 시 덜컹거림 방지)
*   **Footer (#ft)**: 하단 정보 영역.

### 4) 공통 UI 컴포넌트 (Components) - `자주 쓰는 것들`
어느 페이지에서나 쓸 수 있는 버튼, 박스 등입니다.
*   **Buttons**: `.btn-primary` (메인 버튼), `.btn-outline` (테두리 버튼).
*   **Section Title**: 본문 중간중간 나오는 소제목 스타일 (`section h2`).
*   **Breadcrumb**: "Home > 회사소개 > 인사말" 경로 표시 스타일.

### 5) 유틸리티 (Utilities) - `도우미 클래스`
*   `.text-center`, `.mt-50` (마진 탑 50px), `.pc-only` (PC에서만 보임) 등.

## 3. 디자인 토큰 (Design Tokens) - 실제 코드 예시
### Color Palette (Dark Default)
```css
:root {
    /* Base Colors */
    --bg-primary: #121212;   /* 메인 배경 (Dark) <-> (Light: #ffffff) */
    --bg-secondary: #1e1e1e; /* 서브 배경 (Card) <-> (Light: #f5f5f5) */
    
    /* Text Colors */
    --text-primary: #ffffff; /* 기본 텍스트 <-> (Light: #333333) */
    --text-muted: #888888;   /* 보조 텍스트 */
    
    /* Brand Colors */
    --brand-gold: #d4af37;   /* 포인트 컬러 (유지) */
    --brand-blue: #0044cc;   /* 링크/버튼 (유지 or 조정) */
}
```

## 3. 테마 구조 분석 (기존 내용)
그누보드 5 테마는 기본적으로 다음과 같은 파일 실행 순서를 가집니다:
1.  **`head.sub.php`**: HTML `head` 섹션. CSS 및 JS 라이브러리 로드.
2.  **`head.php`**: 웹사이트 상단(Logo, GNB) 및 서브 페이지 레이아웃(Hero Section).
3.  **`index.php`** (메인) 또는 **`skin/*`** (게시판/페이지): 실제 콘텐츠 본문.
4.  **`tail.php`**: 웹사이트 하단(Footer).
5.  **`tail.sub.php`**: HTML 하단 마무리 및 추가 스크립트.

### CSS 로드 방식
- `add_stylesheet('<link rel="stylesheet" href="'.G5_THEME_CSS_URL.'/파일명.css">', 우선순위);` 함수를 사용하여 `head.php`에서 로드하는 것을 권장합니다.

## 2. 테마 골격 (HTML Skeleton) 핵심 역할

테마의 일관성을 유지하기 위해 `#wrapper`, `#container_wr`, `#container`의 역할을 엄격히 구분합니다.

| ID | 명칭 | 핵심 역할 | 디자인 전략 |
| :--- | :--- | :--- | :--- |
| **`body`** | 시스템 바탕 | 사이트 전체 기본 배경 | `--color-bg-dark` 등 테마의 가장 근본 색상 정의. |
| **`#wrapper`** | 디자인 스위치 | 메인/서브 모드 전환 | `_INDEX_` 상수에 따른 **`.sub-layout-light`** 클래스로 전체 톤 제어. |
| **`#container_wr`** | 레이아웃 허브 | 본문 영역 전체 (Full) | 100% 가로너비 배경색/이미지 배치. |
| **`.sub-layout-width-height`** | **표준 가이드 (Hub)** | **컨텐츠 가이드 & 간격** | `max-width: 1200px`, `padding: 60px 20px 50px`로 **정렬 & 여백 통일**. |

> [!IMPORTANT]
> **효율적인 모드 전환 (`_INDEX_` 활용)**
> ```php
> <div id="wrapper" <?php echo (defined('_INDEX_')) ? '' : ' class="sub-layout-light"'; ?>>
> ```
> 위와 같이 `#wrapper`에서 메인/서브를 구분하면 하위 모든 자식 요소의 스타일을 한 번에 제어할 수 있어 유지보수가 매우 유리합니다. 이를 **'디자인 스위치'** 전략이라 합니다.

## 3. 'Corporate' 테마 분석 및 규칙 (Current State)

현재 작업 중인 `corporate` 테마는 **Premium Dark** 컨셉을 유지하며, 다음과 같은 분리 구조를 따릅니다.

### CSS 파일 분할 규칙 (4-File Structure)
- **`default.css`**: 글로벌 디자인 토큰(:root), 리셋, 기본 타이포그래피, 헤더/푸터 공통 스타일.
- **`sub_common.css`**: 서브 페이지 공통 레이아웃 (Hero Section, Breadcrumb).
- **`sub_style.css`**: 서브 페이지 내부 콘텐츠 (테이블, 폼, 버튼 상세 스타일).
- **`mobile.css`**: 모든 미디어 쿼리 및 모바일 전용 UI 스타일.

### 디자인 토큰 (Core Variables)
반드시 `default.css`의 `:root`에 정의된 변수를 사용하여 스타일을 입혀야 합니다.

| 변수 분류 | 변수명 | 기본값 (Corporate) | 용도 |
| :--- | :--- | :--- | :--- |
| **Colors** | `--color-bg-dark` | `#121212` | 전체 배경색 |
| | `--color-bg-panel` | `#1e1e1e` | 섹션/카드 배경 |
| | `--color-accent-gold` | `#d4af37` | 포인트 컬러 (Gold) |
| **Fonts** | `--font-heading` | `'Playfair Display'` | 주요 제목용 세리프체 |
| | `--font-body` | `'Inter'` | 본문용 샌세리프체 |
| **Size** | `--spacing-container`| `1400px` | 최대 콘텐츠 너비 |
| | `--header-height` | `100px` | 헤더 고정 높이 |

## 4. 새로운 테마 제작을 위한 CSS 규칙 (Guidelines)

앞으로 다른 디자인의 테마를 만들 때도 다음 규칙을 준수하여 효율성을 높입니다.

### 1) No Hardcoding (변수 사용 필수)
색상, 폰트 크기, 여백 등을 직접 입력하지 마십시오. 새로운 테마를 만들 때는 변수 값만 바꿔도 전체 분위기가 바뀌도록 설계합니다.

### 2) BEM 네이밍 규칙 준수 (Block-Element-Modifier)
클래스명 충돌을 방지하기 위해 BEM 방식을 권장합니다.
- `block`: `.hero-section`
- `element`: `.hero-section__title`, `.hero-section__image`
- `modifier`: `.hero-section--light`, `.hero-section--full`

### 3) Mobile First & Responsive
- 모든 미디어 쿼리는 한 파일(`mobile.css`) 또는 각 컴포넌트 하단에 모아서 관리합니다.
- 부트스트랩 등 외부 프레임워크 사용 시 테마 커스텀 변수와 충돌하지 않도록 주의합니다.

### 4) Sub Page Hero & Breadcrumb 표준화
- `head.php`에서 `sub_design` 플러그인 데이터를 받아 Hero 이미지를 출력하는 로직을 표준으로 사용합니다.
- 브레드크럼(Home > Depth 1 > Depth 2)은 테마 상관없이 동일한 HTML 구조를 유지하고 CSS로만 디자인을 변경합니다.

## 5. 효율적인 CSS 관리 전략 (Hub & Spoke Model)

테마가 늘어남에 따라 CSS가 스파게티처럼 꼬이는 것을 방지하기 위해 '허브 앤 스포크' 모델을 채택합니다.

### 1) Hub (Central Brain): `default.css`
- **역할**: 테마의 전체 생상, 폰트, 공통 규격 정의.
- **포함 내용**: `:root` 변수, Reset, Global Typography, Header/Footer Layout.
- **관리 원칙**: 모든 디자인 변경은 이 파일의 `:root`에서 시작됩니다.

### 2) Spoke (Specialized Styles): Plugin/Skin CSS
- **역할**: 특정 기능(회사소개, 게시판 리스트 등)에 특화된 레이아웃 정의.
- **관리 원칙**: 
    - 색상/폰트값을 직접 적지 않고 Hub의 변수를 호출합니다.
    - 예: `color: var(--color-accent-gold);`
    - 플러그인 스킨 내부의 `<style>` 태그는 최소화하고, 가능하다면 별도의 `.css` 파일로 분리하여 `add_stylesheet`로 로드합니다.

### 3) 공통 래퍼(Wrapper) 활용
- 모든 서브 페이지는 `#wrapper.sub-layout-light`라는 공통 부모를 가집니다.
- 이를 활용해 서브 페이지 본문 내의 제목(`h1~h6`), 문단(`p`), 링크(`a`)의 기본 스타일을 한 번에 정의하여 코드 중복을 제거합니다.
- 예: `#wrapper.sub-layout-light section h2 { ... }`

### 4) 캐시 버스팅 (Cache Busting)
- CSS 수정 후 즉시 반영되도록 그누보드의 `$G5_CSS_VER` 변수를 활용하거나, 로드 시 `time()` 함수를 파라미터로 붙여 관리합니다.

## 8. 신규 스킨 제작 가이드 (New Skin Development Protocol)
새로운 스킨을 추가할 때는 다음 체크리스트를 반드시 통과해야 합니다.

### [체크리스트]
1.  **[ ] Theme Colors Only**: 색상은 절대 하드코딩하지 않았는가? (Hex/RGB 금지)
2.  **[ ] Font Inheritance**: 폰트 패밀리를 직접 지정하지 않고 상속(`inherit`)하거나 테마 변수(`--font-heading`)를 사용했는가?
3.  **[ ] No Global Styles**: CSS 클래스명이 너무 일반적이지 않은가? (Bad: `.container`, Good: `.ci-container`)
4.  **[ ] Dark Mode Ready**: 배경색은 `--color-bg-dark`, 글자색은 `--color-text-primary`를 사용하여 테마 변경 시 자동 대응 되는가?

> [!WARNING]
> **하드코딩 절대 금지 (Strict Prohibition)**
> ```css
> /* ❌ WRONG */
> .my-skin { background: #000; color: #fff; font-family: 'Gulim'; }
> 
> /* ✅ RIGHT */
> .my-skin {
>     background: var(--color-bg-dark);
>     color: var(--color-text-primary);
>     font-family: var(--font-body);
> }
> ```

## 9. 관리 도구별 분석 및 주의사항 (Analysis by Manager)
현재 구축된 10개의 관리 모듈을 기준으로, 일반 방식과의 차이점과 주의사항을 정리했습니다.

### 1) 테마관리 / 테마설정
*   **일반 방식**: 단순히 스킨 폴더명만 지정 (`basic`, `gallery` 등).
*   **우리 방식**: **`theme.config.php`** 파일과 연동하여 전역 변수(폰트, 색상 등)를 제어하려 했으나, 현재는 주로 **CSS 변수(`default.css`)**에 의존하고 있습니다.
*   **⚠️ 주의**: 관리자 페이지의 설정보다 **`default.css`의 `:root` 변수**가 우선 적용되도록 설계되어 있습니다. 색상을 못 바꿀 때는 관리자가 아닌 CSS 파일을 확인해야 합니다.

### 2) 상단메뉴스킨관리 (`top_menu_manager`)
*   **일반 방식**: `head.php`에 `<ul><li>...</li></ul>` 태그를 직접 코딩.
*   **우리 방식**: 관리자에서 메뉴 구조(1차, 2차)를 DB에서 불러와 **자동으로 렌더링**합니다.
*   **⚠️ 주의**: 메뉴를 추가하려면 소스 수정이 아니라, 관리자 페이지의 **[메뉴설정]**에서 추가해야 합니다. 소스에 직접 넣으면 안 보일 수 있습니다.

### 3) 메인 이미지 관리 (`main_image_manager`)
*   **일반 방식**: `index.php`에 슬라이더 플러그인(Swiper 등) 코드를 직접 삽입.
*   **우리 방식**: 이미지를 업로드하면 자동으로 슬라이더 소스가 생성되어 `index.php`에 주입됩니다.
*   **⚠️ 주의**: 슬라이더의 동작 속도나 효과를 바꾸려면 `index.php`가 아닌 **플러그인 내부의 JS 파일**을 수정해야 합니다.

### 4) 메인 컨텐츠 관리 (`main_content_manager`)
*   **일반 방식**: 최신글 스킨(`latest`)을 단순히 나열.
*   **우리 방식**: '섹션(Section)' 단위로 콘텐츠를 관리하며, 각 섹션마다 **다른 디자인(Style A, B, C)**을 선택할 수 있습니다.
*   **⚠️ 주의**: 새로운 디자인 스타일을 만들려면 플러그인 뷰 폴더에 파일을 추가해야 합니다.

### 5) 서브디자인관리 (`sub_design`)
*   **일반 방식**: 각 페이지 파일 상단에 `$sub_visual_img = '...';` 변수를 직접 선언.
*   **우리 방식**: 메뉴 코드(`me_code`)를 기준으로 **DB에 저장된 이미지**를 불러옵니다.
*   **⚠️ 주의**: 파일명이 아닌 **메뉴 코드**로 매칭되므로, 메뉴 구조가 바뀌면 서브 비주얼 설정도 다시 확인해야 합니다.

### 6) 회사소개 / 온라인문의 / 카피라이트 / 지도 API
*   **일반 방식**: `company.php`, `contact.php`, `tail.php` 파일에 주소와 전화번호를 '하드코딩(직접 입력)'.
*   **우리 방식**: 모든 정보가 **변수(Variable)**화 되어 있습니다. (`{CP_TEL}`, `{CP_ADDR}` 등)
*   **⚠️ 주의**: 주소가 바뀌면 파일을 수정하는 게 아니라, **[카피라이트 관리]** 및 **[회사소개 관리]** 두 곳을 수정해야 전체 사이트에 반영됩니다.
