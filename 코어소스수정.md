# 코어 소스 수정 내역

## 2025-12-19
### 1. 관리자 토큰 처리 로직 수정 (CSRF 방지 강화 및 오류 해결)
**목적**: 관리자 페이지(특히 플러그인 포함)에서 폼 전송 시 토큰 재발급으로 인한 "올바른 방법으로 이용해 주세요" 오류 해결 및 보안 강화.

#### `adm/admin.js`
- **위치**: `submit` 이벤트 핸들러 내부 (약 223라인)
- **수정 내용**: 폼에 이미 유효한 `token`이 있는 경우, 불필요하게 `ajax.token.php`를 호출하여 토큰을 갱신하지 않고 기존 값을 유지하도록 변경.
```javascript
// 이미 토큰 값이 존재하는 경우 새로 생성하거나 덮어쓰지 않음
if ($token.length > 0 && $token.val() !== "") {
    return true;
}
```

#### `adm/ajax.token.php`
- **위치**: 파일 상단
- **수정 내용**: 클라이언트(`admin.js`)로부터 전달받은 `admin_csrf_token_key`가 서버의 키와 일치하는지 검증하는 로직 추가. 비정상적인 경로의 토큰 발급 요청 차단.
```php
$admin_csrf_token_key = isset($_POST['admin_csrf_token_key']) ? $_POST['admin_csrf_token_key'] : '';

if (function_exists('admin_csrf_token_key') && $admin_csrf_token_key !== admin_csrf_token_key(1)) {
    $debug_msg = 'KeyMismatch: Post[' . $admin_csrf_token_key . '] vs Server[' . admin_csrf_token_key(1) . ']';
    die(json_encode(array('error' => $debug_msg, 'url' => G5_URL)));
}
```
