# 🛠️ 보드 스킨 관리 표준 (Board Skin Management Standard)

> **"표준을 준수하지 않는 스킨은 테마의 일부가 될 수 없습니다."**
> 이 문서는 테마의 **`default.css`**를 기반으로 게시판 스킨을 개발/유지보수할 때 반드시 따라야 할 **기술 표준(Technical Standard)**을 정의합니다.

---

## 🏗️ 1. 스킨 개발 4단계 프로세스 (Development Lifecycle)

스킨 제작 시 아래 4단계 프로세스를 순차적으로 적용하여 기능 누락을 원천 차단하십시오.

### (0) 공통 표준 및 스타일 가이드 (Common Rules & Style Guide)
*   **식별자 (Identity)**:
    *   폴더명: `skin_타입명` (예: `skin_magazine`)
    *   게시판 ID: `corporate_타입명` (예: `corporate_portfolio`)
*   **Context 유지**: 모든 내부 링크(`href`)에 `me_code`(메뉴)와 `sca`(카테고리) 파라미터를 반드시 이어붙여야 합니다. (`$qstr` 활용 권장)
*   **CSS & JS Policy**:
    *   **Scoped Styling**: 모든 스타일은 `.skin-container` 등으로 래핑하여 외부 영향 최소화.
    *   **Hidden Input Validation**: `display:none` 처리된 입력 필드(예: 커스텀 카테고리)에는 `required` 속성을 금지하며, 반드시 JS (`onsubmit`)로 유효성을 검사해야 합니다.
    *   **JS Module**: 복잡한 로직은 별도 `script.js`로 분리 권장.
*   **Layout 상속**: 레이아웃 래퍼로 `.skin-container` (1360px) 사용 필수.
*   **Buttons 상속**: 자체 버튼 스타일 대신 `.mc-btn` (기본), `.mc-btn-primary`, `.mc-btn-outline` 클래스 사용 권장.
*   **Inputs 상속**: `style.css`에서 테마 변수를 매핑 (`input.frm_input { border: 1px solid var(--color-border); ... }`).
*   **Typography (한글 고딕 표준)**:
    *   **Rule**: 한글 텍스트(제목 포함)는 가독성과 심미성을 위해 반드시 **고딕체(`var(--font-body)`)**를 사용합니다. 명조체는 영문 디자인 요소에만 제한적으로 사용합니다.
    *   **Inheritance**: `font-family: var(--font-body);` 명시 필수.

### 0.1 디자인 시스템 참조 (Design System Reference)
보드 스킨 제작 시 **현재 활성화된 테마(Active Theme)**의 **`css/default.css`**에 정의된 아래 전역 변수를 반드시 사용해야 합니다. (하드코딩 금지)

#### 🎨 색상 (Colors)
| 변수명 | 설명 | 값(Light) |
| :--- | :--- | :--- |
| **`var(--color-bg-dark)`** | 기본 배경색 | `#F3F3F3` |
| **`var(--color-brand-etsy)`** | 브랜드 컬러 (강조) | `#F1641E` |
| **`var(--color-accent-gold)`** | 서브 강조색 | `#D4AF37` |
| **`var(--color-text-primary)`** | 본문 텍스트 | `#333333` |
| **`var(--color-border)`** | 입력폼/구분선 | `#DDDDDD` |

#### 📐 간격 및 레이아웃 (Layout)
| 변수명 | 설명 | 값 |
| :--- | :--- | :--- |
| **`var(--spacing-container)`** | 게시판 최대 너비 | **1360px** |
| **`var(--header-height)`** | 헤더 높이 | `100px` |
| **`var(--header-anchor-offset)`** | 앵커 스크롤 오프셋 | `130px` |

#### 🔤 폰트 (Typography)
| 변수명 | 설명 | 비고 |
| :--- | :--- | :--- |
| **`var(--font-body)`** | 본문/제목 (고딕) | 한글 표준 |
| **`var(--font-header)`** | 영문 제목 (명조) | 영문 디자인용 |

### (1) 리스트 페이지 (List Page)
*   **레이아웃**: `Masonry` 또는 `Grid` 스타일을 기본으로 하며, 반응형(`mobile`) 처리를 우선합니다.
*   **책갈피(Anchor) 표준**:
    *   **View to List**: 상세 화면에서 목록으로 돌아활 때는 반드시 **`#bo_list`** 앵커를 사용합니다. (`list.skin.php` 최상단에 `id="bo_list"` 부여 및 `scroll-margin-top: 130px` 적용)
    *   **Write to List (Cancel)**: 글쓰기 취소 시에도 `#bo_list`로 이동하여 기존 위치를 유지합니다.
*   **필터링**: 카테고리 탭 클릭 시 `sfl/stx` 검색 방식으로 구현하여 매칭 정확도를 높입니다. (공백/인코딩 이슈 방지)

### (2) 레이어 갤러리 (Layer Viewer - Premium)
*   **Split Layout**: 프리미엄 스킨(Magazine 등)은 60(갤러리):40(정보) 스플릿 레이아웃을 기본으로 합니다.
*   **UX**: 이미지 클릭 시 페이지 이동 없이 **모달(Modal)**을 띄우는 것을 권장합니다.
*   **Navigation**: 이전/다음 화살표는 모달 박스 **바깥(ViewPort 기준 Fixed)**에 배치합니다.
*   **Context**: 모달 내부에 '상세보기(Read More)' 버튼을 두어 `me_code`가 포함된 개별 뷰 페이지로 연결합니다.

### (3) 뷰어 페이지 (View Page / General)
> [!TIP]
> **"Information Layering: Separate Data from Meta."**

*   **배치 (Asymmetric Grid)**:
    - 본문(`article-main`): 좌측 또는 중앙 배치 (최대 너비 700~800px 권장).
    - 메타 정보(`meta-sidebar`): 우측 **Sticky Sidebar** 배치. (작성일, 카테고리 등)
*   **Typography Hierarchy**:
    - **Article Lead**: 본문 첫 문단은 큰 폰트/볼드 처리하여 요약 강조.
*   **Wrapper Standard**: `view.skin.php`는 `skin-container` 하나로 정리합니다.
*   **Anchor**: 상세 페이지 최상단 래퍼에 `id="bo_view"`를 부여하고 `scroll-margin-top`을 적용합니다.

### (4) 등록/수정 페이지 (Write Page)
*   **Structure**: `<section id="bo_w" class="skin-container">` 구조 엄수.
*   **Zero-Header Policy**: `write.skin.php` 내부에 "New Entry" 등의 하드코딩된 헤더를 **절대 포함하지 않습니다**.
*   **Logic (JS)**: 다단계 카테고리는 표준 모듈 `theme/corporate_light/js/category_cascade.js` 사용.
*   **Flow (Redirect)**: `write_update.tail.skin.php`를 사용하여 저장 후 **목록(List)으로 이동**시키고, `me_code`와 `bo_list` 앵커를 유지합니다.

## 🛠️ 2. 트러블슈팅 가이드 (Troubleshooting)

### Q1. 카테고리 필터 결과가 0건입니다.
*   **원인 1 (Anchor 위치)**: 앵커(`#bo_list`)가 URL 파라미터 중간에 끼어있으면 브라우저가 뒷부분을 무시합니다. 앵커는 무조건 URL **맨 끝**에 와야 합니다.
*   **원인 2 (매칭 불일치)**: `sca` 파라미터는 완전 일치만 허용하므로, 공백 하나만 달라도 실패합니다. `sfl=ca_name&stx=분류명` 검색 방식으로 전환하십시오.

### Q2. 스크롤이 헤더에 가립니다.
*   **해결**: `style.css`에 아래 코드를 추가하여 헤더 높이만큼 여백을 확보하십시오.
    ```css
    #bo_list, #bo_view, #bo_w {
        scroll-margin-top: 130px; /* 또는 var(--header-anchor-offset) */
    }
    ```

### Q3. 게시판 설정이 오염되었습니다 (알 수 없는 숫자 붙음).
*   **해결**: `fix_category.php` 스크립트를 작성하여 `bo_category_list` 컬럼을 강제 초기화하십시오.
