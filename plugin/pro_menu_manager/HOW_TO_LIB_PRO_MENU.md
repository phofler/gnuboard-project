# Pro Menu Manager 개발 및 유지보수 가이드 (lib.md)

이 문서는 Pro Menu Manager 플러그인의 핵심 로직, 인코딩 표준 및 유지보수 지침을 기록합니다.

## 1. 핵심 로직: 메뉴 코드 생성 (`ma_code`)

메뉴의 계층 구조와 정렬을 위해 2자리 단위의 `ma_code`를 사용합니다.

- **함수**: `get_next_pro_menu_code($parent, $max_code)` in `lib.php`
- **규칙**:
  1. **순차 증가 (+1)**: 기존 10단위 증가 방식에서 발생하던 오버플로(90+10=100)를 방지하기 위해 1단위로 증가합니다. (예: `1010, 1011, 1012...`)
  2. **알파벳 확장**: 숫자가 99를 초과할 경우 `a0, a1... z9` 형식의 알파벳 코드를 생성하여 항상 2자리를 유지합니다.
  3. **계층 구조**: 부모 코드 뒤에 새로운 2자리를 덧붙여 하위 메뉴를 생성합니다. (예: 부모 `10` -> 자식 `1010, 1011`)

## 2. 파일 인코딩 표준

윈도우 환경과 GnuBoard(EUC-KR/UTF-8) 간의 특수문자 및 한글 깨짐을 방지하기 위한 절대 규칙입니다.

- **표준 형식**: **BOM 없는 UTF-8 (UTF-8 without BOM)**
- **수정 지침**: PHP `alert()` 메시지나 한글 주석이 포함된 파일을 수정할 때는 반드시 인코딩을 확인해야 합니다.
- **저장 팁**: PowerShell이나 Python 등을 이용해 저장할 때 명시적으로 `UTF8Encoding($false)`를 지정하십시오.

## 3. 서브 디자인(Sub Design) 연동

브레드크럼(LNB) 및 서브 비주얼과의 연동 지침입니다.

- **테이블 매핑**: `g5_plugin_sub_design_groups` 테이블의 `sd_menu_table` 컬럼에 사용할 메뉴 테이블 접미사를 지정합니다. (기본값: `default` -> `g5_write_menu_pdc`)
- **이름 동기화**: `plugin_sub_design_items` 테이블의 `me_name`이 비어 있어야 Pro Menu Manager의 최신 이름을 실시간으로 반영합니다.

## 4. 장애 복구 (Troubleshooting)

- **메뉴 순서 꼬임**: `ma_code`가 5자리(예: `20100`)로 생성된 데이터가 있는지 확인하고, 이를 4자리 순차 번호로 직접 수정하십시오.
- **한글 깨짐(모지바케)**: 파일을 열어 `iconv`나 적절한 에디터로 BOM 없는 UTF-8로 다시 저장하십시오.
