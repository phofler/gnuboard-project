# 그누보드 프로젝트 개발 원칙 (Project Rules)

이 문서는 프로젝트 진행 시 반드시 준수해야 할 원칙과 유의사항을 기록합니다.

## 1. 코어 보존 (Core Integrity)
- **원칙**: `/bbs`, `/lib`, `/adm` 등 그누보드 핵심 코어 파일은 직접 수정하지 않습니다. 업데이트 시 충돌을 방지하기 위함입니다.
- **방법**:
  - 기능 변경은 `/extend` 폴더 내의 Hook 파일을 사용합니다.
  - 디자인 변경은 `/theme/corporate` 테마 폴더 내에서만 진행합니다.
  - 관리자 기능 확장이 필요할 경우, 가능한 `/theme` 내의 스킨을 활용하거나 별도 메뉴를 추가하는 방식을 권장합니다.

## 2. 데이터베이스 호환성 (Database Compatibility)
- **이슈**: MySQL Strict Mode 환경에서 `datetime` 등의 컬럼에 잘못된 기본값(예: `0000-00-00`)이 들어가면 오류가 발생합니다.
- **원칙**: 테이블 생성(`CREATE TABLE`)이나 변경 시, `NOT NULL` 컬럼에는 반드시 명시적인, 유효한 `DEFAULT` 값을 지정해야 합니다.
  - 예: `datetime NOT NULL DEFAULT '0000-00-00 00:00:00'` (X) -> `datetime NOT NULL DEFAULT CURRENT_TIMESTAMP` 또는 유효한 날짜 (O)

## 3. 환경 및 경로 (Environment & Paths)
- **파일**: 리눅스 서버 배포를 고려하여 파일명은 모두 **소문자**로 작성합니다.
- **경로**: 절대 경로보다는 그누보드 상수(`G5_THEME_PATH` 등)를 활용한 상대 경로를 사용합니다.

## 4. 보안 (Security)
- **입력 검증**: 사용자 입력 데이터(`$_POST`, `$_GET`)는 `clean_xss_tags()` 등 그누보드 내장 함수를 사용하여 검증 후 저장합니다.
- **민감 정보**: SMTP 앱 비밀번호, API 키 등은 소스코드에 하드코딩하지 않고 변수나 별도 설정 파일로 관리합니다.

## 5. 디버깅 및 로깅 (Debugging)
- **개발 중**: `config.php`의 `G5_DISPLAY_SQL_ERROR`를 활용하되,
- **배포 시**: 반드시 해당 옵션을 비활성화(False)하여 정보 노출을 막습니다.

## 6. 프로그램 추가 및 확장 (Program Addition)
- **원칙**: 새로운 프로그램이나 기능을 추가할 때는 `plugin/` 또는 `extend/` 폴더를 적극 활용합니다.
  - **plugin/**: 독립적인 기능모듈이나 외부 라이브러리 연동 시 사용합니다.
  - **extend/**: 그누보드 실행 시 자동으로 로드되어야 하는 훅(Hook)이나 코어 로직 확장 시 사용합니다.
