# 카피라이트 매니저 복구 및 작동 원리 가이드 (HOW_TO_RECOVER_COPYRIGHT_MANAGER.md)

## 🚨 1. 작동 원리 (Working Principle) - 필독
이 모듈은 사용자 디자인의 자유도와 자동화의 편의성을 동시에 제공하기 위해 **이중 출력 구조**를 가집니다.

1.  **[우선순위 1] 자유 에디터 (Editor Priority)**
    *   관리자 페이지 하단의 '자유 에디터'에 내용이 입력되어 있으면, **스킨의 기본 템플릿을 무시**하고 에디터 내용을 출력합니다.
    *   이때, 에디터 내의 텍스트 중 `{addr}`, `{company}`, `{ceo}`, `{bizno}` 등의 키워드는 관리자 설정값으로 자동 치환됩니다.
    *   **주의**: 에디터에 주소를 글자 그대로(예: 파주시...) 입력하면 관리자에서 주소를 바꿔도 실사이트에는 옛날 주소가 계속 나옵니다. 반드시 **치환자**를 사용하십시오.

2.  **[우선순위 2] 표준 스킨 (Fallback Template)**
    *   에디터 내용이 **완전히 비어 있을 때**만 작동합니다.
    *   Style A~D에서 정의한 고정 레이아웃(Grid, Flex)에 따라 정보를 정렬하여 보여줍니다.

---

## 🛠 2. 주요 치환자 (Placeholders)
에디터에서 자유롭게 배치할 때 다음 키워드를 사용하면 관리자 설정값과 실시간 연동됩니다.

*   `{company}` : 상호 (값) / `{company_name}` : 상호 (라벨)
*   `{ceo}` : 대표자 (값) / `{ceo_name}` : 대표자 (라벨)
*   `{bizno}` : 사업자번호 (값) / `{bizno_name}` : 사업자번호 (라벨)
*   `{addr}` : 주소 / `{tel}` : 전화 / `{fax}` : 팩스 / `{email}` : 이메일
*   `{logo}` : 하단 로고 이미지 / `{copyright}` : 카피라이트 문구

---

## 🔄 3. 데이터 동기화 문제 해결 (Sync Fix)
만약 관리자에서 정보를 바꿨는데 실사이트에 옛날 정보가 나온다면, 에디터에 옛날 글자가 직접 타이핑되어 있는지 확인하십시오.

**해결 방법**:
1. 에디터의 옛날 글자를 지우고 위 치환자들로 교체합니다.
2. 또는 `fix_content_sync.php` 도구를 실행하여 자동으로 치환자로 바꿉니다.
